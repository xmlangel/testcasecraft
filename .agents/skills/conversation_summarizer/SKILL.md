# Conversation Summarizer Skill

진행 중인 대화 내용을 요약하여 `docs/` 디렉토리에 Markdown 파일로 기록하는 스킬입니다.

## Metadata

- **Name**: conversation_summarizer
- **Description**: 현재 세션의 주요 논의 사항, 결정 사항, 작업 내역을 정리하여 문서화합니다.
- **Output Directory**: `docs/`

## Usage

대화 내용이 복잡해지거나, 특정 기능 구현이 완료되었을 때 인공지능에게 이 스킬을 사용하여 내용을 정리해달라고 요청할 수 있습니다.

### 호출 예시

- "지금까지 대화한 내용을 `docs/`에 정리해줘."
- "이 기능을 구현하면서 나눈 논의를 요약해서 문서로 만들어줘."

## 요약 가이드라인

생성되는 문서(`docs/*.md`)는 다음과 같은 구조를 권장합니다:

1.  **제목**: `[YYYY-MM-DD] 주제 요약`
2.  **개요**: 이번 대화의 핵심 목표 및 배경
3.  **주요 논의 사항**: 결정된 설계 방향, 기술 스택, 사용자 피드백 등
4.  **작업 내역**: 구현된 파일, 변경된 코드 위치, 수정된 버그
5.  **검증 결과**: 테스트 내역 및 성공 여부
6.  **향후 계획**: (데이터가 있을 경우) 다음에 진행할 작업들

## 파일명 규칙

- 대화의 핵심 주제를 반영하여 `docs/summary_topic_name.md` 형태로 저장합니다.
- 예: `docs/summary_google_login_impl.md`
