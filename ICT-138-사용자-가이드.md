# ICT-138: 테스트케이스 스프레드시트 형태 일괄 입력 기능 사용자 가이드

## 🚀 새로운 기능 소개

기존의 개별 폼 입력 방식을 유지하면서, Excel과 유사한 스프레드시트 환경에서 여러 테스트케이스를 한 번에 일괄 입력할 수 있는 **하이브리드 입력 시스템**이 추가되었습니다.

## 📊 주요 특징

### ✅ 하이브리드 시스템
- **개별 폼 모드**: 기존 방식 유지 (상세 입력, 무제한 스텝)
- **스프레드시트 모드**: 신규 기능 (빠른 일괄 입력, 최대 3개 스텝)
- **원클릭 전환**: 언제든지 모드 변경 가능, 데이터 손실 없음

### 🎯 성능 개선
- **80% 시간 단축**: 100개 테스트케이스 등록 시간 5시간 → 1시간
- **50개+ 동시 편집**: 한 화면에서 대량 데이터 처리
- **실시간 검증**: 입력 즉시 유효성 검사 및 피드백

## 🔧 사용 방법

### 1. 입력 모드 선택
1. **테스트케이스 탭**으로 이동
2. 상단의 **입력 모드 토글**에서 원하는 모드 선택
   - **개별 폼**: 상세한 테스트케이스 작성
   - **스프레드시트**: 빠른 일괄 입력

### 2. 스프레드시트 모드 사용법

#### 📝 컬럼 구조
| 컬럼 | 설명 | 필수 여부 |
|------|------|-----------|
| 이름 | 테스트케이스 이름 | ✅ 필수 |
| 설명 | 테스트케이스 설명 | 선택 |
| 사전조건 | 테스트 실행 전 조건 | 선택 |
| 예상결과 | 전체 테스트 예상 결과 | 선택 |
| Step 1-3 | 테스트 실행 단계 | 선택 |
| Expected 1-3 | 각 단계별 예상 결과 | 선택 |

#### ⌨️ 편집 기능
- **셀 편집**: 셀 클릭 후 직접 입력
- **네비게이션**: Tab/Enter로 다음 셀 이동
- **복사/붙여넮기**: Ctrl+C/V 지원
- **행 추가**: "행 추가" 버튼으로 5행씩 추가
- **새로고침**: 원본 데이터로 복원

#### 💾 저장 기능
- **실시간 추적**: 변경 상태 실시간 표시
- **일괄 저장**: "일괄 저장" 버튼으로 모든 변경사항 저장
- **자동 처리**: 신규 추가 + 기존 업데이트 자동 판별

### 3. 모드 전환
- **언제든지 전환 가능**: 작업 중 모드 변경 가능
- **데이터 보존**: 모드 전환 시 편집 중인 데이터 유지
- **경고 표시**: 저장되지 않은 변경사항 알림

## 🎨 사용자 경험 개선

### 📱 직관적 UI
- **상태 표시**: 현재 행 수, 변경 상태 표시
- **사용법 안내**: Excel 사용자를 위한 친숙한 가이드
- **에러 처리**: 명확한 오류 메시지 및 해결 방법 제시

### ⚡ 성능 최적화
- **지연 로딩**: 필요한 데이터만 로드
- **메모리 효율**: 대용량 데이터 최적 처리
- **응답성**: 실시간 피드백으로 빠른 작업 환경

## 🔄 워크플로우 예시

### 시나리오 1: 새 테스트케이스 대량 등록
1. **스프레드시트 모드** 선택
2. 첫 번째 행에 테스트케이스 정보 입력
3. Tab으로 다음 셀 이동하며 빠르게 입력
4. "행 추가"로 추가 행 확보
5. 모든 데이터 입력 후 "일괄 저장" 클릭

### 시나리오 2: 기존 테스트케이스 수정
1. **개별 폼 모드**에서 특정 테스트케이스 선택
2. 상세 수정 후 저장
3. 필요시 **스프레드시트 모드**로 전환하여 전체 리스트 확인

### 시나리오 3: 하이브리드 작업
1. **스프레드시트 모드**로 빠른 초안 작성
2. **개별 폼 모드**로 전환하여 상세 내용 보완
3. 복잡한 테스트케이스는 폼에서, 간단한 케이스는 스프레드시트에서 작업

## ⚠️ 주의사항

### 📋 제약사항
- **스프레드시트 모드**: 최대 3개 스텝만 지원
- **복잡한 케이스**: 4개 이상 스텝 필요시 개별 폼 사용 권장
- **네트워크**: 일괄 저장 시 안정적인 네트워크 연결 필요

### 💡 팁
- **백업**: 대량 작업 전 기존 데이터 백업 권장
- **단계별 저장**: 50개씩 나누어 저장하여 안정성 확보
- **검증**: 저장 후 개별 폼에서 최종 검토

## 🆘 문제 해결

### 자주 발생하는 문제
1. **저장 실패**: 네트워크 연결 확인, 브라우저 새로고침
2. **데이터 손실**: "새로고침" 버튼으로 원본 데이터 복원
3. **모드 전환 오류**: 페이지 새로고침 후 재시도

### 지원 문의
- **기술 지원**: 시스템 관리자 문의
- **사용법 문의**: 사용자 매뉴얼 참조
- **기능 개선 제안**: 개발팀 피드백 채널 활용
