# Commit Message Template

## Format
[EN] English summary / [KO] Korean summary

## Examples
- [EN] Add user authentication feature / [KO] 사용자 인증 기능 추가
- [EN] Fix login validation bug / [KO] 로그인 검증 버그 수정
- [EN] Update dashboard UI components / [KO] 대시보드 UI 컴포넌트 업데이트
- [EN] Implement Jira integration / [KO] Jira 연동 기능 구현
- [EN] Refactor test case management / [KO] 테스트 케이스 관리 리팩토링

## Guidelines
- Keep English summary concise and descriptive
- Korean summary should be natural and clear
- Use present tense for both languages
- Include the main action and affected area
- Maximum length: 120 characters total
