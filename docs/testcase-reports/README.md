# PG17 실패 케이스 → 테스트 결과 리포트

`~/Downloads/pg17pg_fail_cases.html`(AgensSQL 17.10.0-47 PG-mode 실패 21건)을
TestCaseCraft 앱의 **테스트 결과 리포트 출력** 형식으로 변환한 산출물.

## 산출물

| 파일 | 설명 |
|------|------|
| `PG17_FAIL_CASES.html` | 앱 리포트 레이아웃(헤더 + KPI 카드 4 + 상태 브레이크다운 + 케이스 요약/상세 + 푸터). **라이트/다크 테마** 지원 |
| `PG17_FAIL_CASES.md` | 동일 내용의 Markdown TC 문서(요약 표 + 케이스별 단계·diff) |
| `pg17_fail_cases.json` | 파싱된 구조화 데이터(중간 산출물, 재가공용) |
| `parse_fail_cases.py` | 변환기 |

## 앱 리포트와 동일하게 맞춘 점

- **레이아웃**: `TestResultExportDialog.jsx` 의 PDF export(`exportAsPdf`) 구조를 그대로 차용
  - KPI 카드: 총 테스트 / 실행률 / 성공률 / JIRA 연동
  - 상태 브레이크다운: 성공 / 실패 / 차단 / 미실행
  - 푸터: `Generated by TestCaseCraft - Professional QA Reporting`
- **통계**: `computeStatisticsSummary()` 와 동일한 율 계산(실행률 = 실행/전체, 성공률 = 통과/실행)
- **색상 토큰**: MUI 표준(primary `#1976D2`, success `#2E7D32`, error `#D32F2F`, warning `#ED6C02`, info `#0288D1`)
- **테마**: `localStorage["themeMode"]`(light/dark)를 읽어 **사용자가 앱에서 선택한 테마를 유지**.
  우상단 토글로 전환하면 선택이 저장된다. (`ThemeContext.jsx` 의 `data-theme` 규칙과 동일)

## 재생성

```bash
python3 docs/testcase-reports/parse_fail_cases.py ~/Downloads/pg17pg_fail_cases.html docs/testcase-reports
```

입력 HTML 구조(`section.case` × N, 각 3필드: 설명/단계/예상-vs-실제 diff)만 같으면
다른 실행 리포트에도 그대로 재사용 가능.
