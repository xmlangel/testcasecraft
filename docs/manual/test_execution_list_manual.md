# 테스트 실행 리스트 매뉴얼

## 1. 테스트 실행 리스트 개요

테스트 실행 리스트는 시스템 내에서 수행된 모든 테스트 실행 기록을 한눈에 확인하고 관리할 수 있는 기능입니다. 특정 테스트 플랜에 대한 실행 이력뿐만 아니라, 전체 프로젝트 또는 특정 기간 동안의 모든 테스트 실행 내역을 조회할 수 있습니다. 이를 통해 테스트 활동의 전반적인 흐름을 파악하고, 과거 실행 결과를 쉽게 참조할 수 있습니다.

### 주요 목적:
- **실행 이력 조회**: 과거에 수행된 모든 테스트 실행의 기록을 확인합니다.
- **진행 상황 모니터링**: 현재 진행 중이거나 완료된 테스트 실행의 상태를 파악합니다.
- **결과 분석 및 비교**: 특정 시점의 테스트 결과를 다른 시점의 결과와 비교하여 추이 분석에 활용합니다.
- **재실행 및 관리**: 필요에 따라 과거 실행을 참조하여 재실행하거나 관련 정보를 관리합니다.

## 2. 테스트 실행 리스트 접근 및 조회

### 2.1. 리스트 접근

1.  **메뉴 접근**: 애플리케이션의 내비게이션 메뉴에서 '테스트 실행' 또는 '실행 이력'과 같은 항목을 클릭하여 테스트 실행 리스트 화면으로 이동합니다.
2.  **테스트 플랜을 통한 접근**: 특정 테스트 플랜 상세 화면에서 해당 플랜과 관련된 실행 이력 리스트로 바로 이동할 수도 있습니다.

### 2.2. 리스트 필터링 및 검색

테스트 실행 리스트는 다양한 조건으로 필터링하고 검색할 수 있습니다.

-   **프로젝트별 필터링**: 특정 프로젝트에 속한 테스트 실행만 조회합니다.
-   **테스트 플랜별 필터링**: 특정 테스트 플랜에 대한 실행 이력만 조회합니다.
-   **기간별 필터링**: 특정 시작일과 종료일 사이에 수행된 실행만 조회합니다.
-   **실행자별 필터링**: 특정 사용자가 수행한 실행만 조회합니다.
-   **상태별 필터링**: '진행 중', '완료', '취소됨' 등 실행 상태별로 필터링합니다.
-   **키워드 검색**: 실행 이름이나 설명에 포함된 키워드로 검색합니다.

## 3. 테스트 실행 상세 정보 확인

리스트에서 특정 테스트 실행 항목을 클릭하면 해당 실행에 대한 상세 정보를 확인할 수 있습니다.

-   **실행 개요**: 실행 이름, 시작/종료 시간, 실행자, 총 테스트 케이스 수, 성공/실패/블록됨 등의 요약 정보.
-   **포함된 테스트 케이스 목록**: 해당 실행에 포함된 모든 테스트 케이스와 각 케이스의 최종 실행 결과.
-   **개별 테스트 케이스 결과 상세**: 각 테스트 케이스를 클릭하여 해당 케이스의 실행 단계별 결과, 코멘트, 첨부 파일, 연결된 결함 등의 상세 정보를 확인합니다.
-   **실행 로그**: 실행 과정에서 기록된 시스템 로그 또는 사용자 활동 로그.

## 4. 테스트 실행 리스트 활용

### 4.1. 진행 상황 모니터링

현재 진행 중인 테스트 실행의 상태를 주기적으로 확인하여 전체 테스트 진행 상황을 파악하고, 병목 현상이나 지연 요소를 식별할 수 있습니다.

### 4.2. 회귀 테스트 및 재실행 참조

과거의 안정적인 버전 또는 특정 기능에 대한 테스트 실행 결과를 참조하여 새로운 변경 사항에 대한 회귀 테스트 계획을 수립하거나, 유사한 테스트를 재실행할 때 활용할 수 있습니다.

### 4.3. 보고서 생성

테스트 실행 리스트에서 필터링된 데이터를 기반으로 다양한 형태의 보고서(예: 주간 테스트 보고서, 특정 기능 테스트 완료 보고서)를 생성하여 공유할 수 있습니다.

## 5. 주요 기능

-   **포괄적인 실행 이력**: 모든 테스트 실행 기록을 통합 관리합니다.
-   **강력한 필터링 및 검색**: 다양한 조건으로 원하는 실행 기록을 빠르게 찾을 수 있습니다.
-   **상세 정보 제공**: 각 실행 및 포함된 테스트 케이스의 상세 결과를 제공합니다.
-   **보고서 연동**: 리스트 데이터를 기반으로 보고서를 생성하여 테스트 현황을 공유합니다.
