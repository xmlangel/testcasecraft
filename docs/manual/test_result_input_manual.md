# 테스트 결과 입력 매뉴얼

## 1. 테스트 결과 입력 개요

테스트 결과 입력은 테스트 케이스를 실행한 후, 해당 테스트 케이스의 수행 결과를 시스템에 기록하는 과정입니다. 이 과정은 테스트의 성공/실패 여부를 명확히 하고, 발견된 결함에 대한 정보를 연결하며, 테스트 진행 상황을 정확하게 반영하는 데 매우 중요합니다. 정확하고 상세한 결과 입력은 소프트웨어 품질 관리의 핵심입니다.

### 주요 목적:
- **테스트 진행 상황 업데이트**: 각 테스트 케이스의 현재 상태를 반영하여 전체 테스트 진행률을 계산합니다.
- **결함 추적성 확보**: 실패한 테스트 케이스와 관련된 결함을 명확히 연결하여 추적성을 높입니다.
- **품질 지표 산출**: 성공률, 실패율 등 다양한 품질 지표를 산출하는 데 필요한 데이터를 제공합니다.
- **의사 결정 지원**: 테스트 결과를 바탕으로 제품의 출시 여부나 추가 개발 필요성 등 중요한 의사 결정을 지원합니다.

## 2. 테스트 결과 입력 절차

테스트 케이스 실행 화면 또는 테스트 플랜 실행 화면에서 각 테스트 케이스에 대한 결과를 입력할 수 있습니다.

### 2.1. 결과 입력 화면 접근

1.  **테스트 케이스 실행 중**: 테스트 플랜을 통해 실행 중인 경우, 각 테스트 케이스 옆에 있는 결과 입력 버튼을 클릭하거나, 테스트 케이스를 클릭하여 상세 결과 입력 화면으로 이동합니다.
2.  **개별 테스트 케이스 상세**: 개별 테스트 케이스 상세 화면에서 '결과 입력' 또는 '실행 기록' 버튼을 클릭합니다.

### 2.2. 결과 상태 선택

가장 먼저 해당 테스트 케이스의 최종 실행 상태를 선택합니다. 일반적으로 다음과 같은 상태가 제공됩니다.

-   **성공 (Pass)**: 테스트 케이스의 모든 기대 결과가 충족되었을 때 선택합니다.
-   **실패 (Fail)**: 테스트 케이스의 기대 결과 중 하나라도 충족되지 않았을 때 선택합니다. (이 경우 결함 보고가 필요합니다.)
-   **블록됨 (Blocked)**: 테스트 환경 문제, 선행 조건 미충족, 다른 결함 등으로 인해 테스트 케이스를 더 이상 진행할 수 없을 때 선택합니다.
-   **건너뜀 (Skipped)**: 특정 사유로 인해 테스트 케이스를 실행하지 않고 넘어갈 때 선택합니다.

### 2.3. 상세 정보 입력

결과 상태 선택 후, 필요에 따라 다음과 같은 상세 정보를 추가로 입력합니다.

-   **코멘트/비고**: 테스트 실행 중 관찰된 특이사항, 실패 원인, 재현 방법, 환경 정보 등 상세한 내용을 자유롭게 기록합니다. 이는 향후 결함 분석이나 회귀 테스트 시 중요한 참고 자료가 됩니다.
-   **첨부 파일**: 테스트 결과를 증명할 수 있는 스크린샷, 동영상, 로그 파일 등을 첨부합니다. 특히 실패한 경우, 문제 상황을 명확히 보여주는 증거 자료를 첨부하는 것이 중요합니다.
-   **결함 연결**: '실패' 상태를 선택한 경우, 기존에 등록된 결함(버그)을 연결하거나, 새로운 결함을 즉시 생성하여 연결할 수 있습니다. 결함과의 연결은 추적성을 확보하는 데 필수적입니다.
-   **실행자**: 누가 이 테스트를 실행하고 결과를 입력했는지 자동으로 기록되거나 수동으로 선택할 수 있습니다.
-   **실행 시간**: 테스트가 실행된 시간과 결과가 입력된 시간이 자동으로 기록됩니다.

### 2.4. 결과 저장

모든 정보 입력이 완료되면 '저장' 또는 '완료' 버튼을 클릭하여 결과를 시스템에 반영합니다. 저장된 결과는 테스트 진행 상황 및 보고서에 즉시 반영됩니다.

## 3. 결과 입력 시 유의사항

-   **정확성**: 테스트 결과를 객관적이고 정확하게 입력해야 합니다. 주관적인 판단보다는 실제 관찰된 사실을 기반으로 합니다.
-   **상세함**: 특히 실패 또는 블록된 경우, 문제 재현에 필요한 모든 정보를 상세하게 기록합니다.
-   **증거 자료**: 스크린샷이나 로그 등 증거 자료를 첨부하여 결과의 신뢰성을 높입니다.
-   **즉시성**: 테스트 실행 직후 바로 결과를 입력하여 정보의 누락이나 혼동을 방지합니다.

## 4. 주요 기능

-   **다양한 결과 상태**: 테스트 상황에 맞는 정확한 상태를 선택할 수 있습니다.
-   **상세 코멘트 및 첨부**: 문제 재현 및 분석에 필요한 모든 정보를 기록하고 첨부할 수 있습니다.
-   **결함 관리 연동**: 실패한 테스트 케이스를 결함과 직접 연결하여 효율적인 결함 관리를 지원합니다.
-   **자동 기록**: 실행자, 실행 시간 등 기본 정보가 자동으로 기록되어 편리합니다.
