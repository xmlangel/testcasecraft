# API 토큰 관리 매뉴얼

## 1. 개요

이 문서는 TestCaseCraft의 **서비스 API 토큰(Service API Token)** 발급 및 관리 방법을 안내합니다.
API 토큰은 외부 시스템(예: Jira Forge 앱, CI/CD 파이프라인)이 TestCaseCraft API에 인증된 요청을 보낼 때 사용합니다.

> **주의**: 발급된 토큰은 발급 시 **단 한 번만** 전체 값이 표시됩니다. 창을 닫으면 다시 확인할 수 없으므로 즉시 안전한 곳에 보관하세요.

---

## 2. 토큰 관리 화면 접근

1. 우측 상단의 **사용자 아바타 아이콘** 클릭
2. **"프로필"** 메뉴 선택
3. 프로필 다이얼로그에서 **"API 토큰"** 탭 클릭

---

## 3. 주요 기능

### 3.1. 새 API 토큰 발급

1. **토큰 이름** 입력 필드에 식별 가능한 이름 입력
   - 예: `Jira Integration`, `CI/CD Pipeline`, `External Monitor`
2. **"발급"** 버튼 클릭
3. 발급된 토큰이 팝업으로 표시됨
4. **"토큰 전체 복사"** 버튼 또는 우측 복사 아이콘으로 클립보드에 복사
5. 팝업의 **"확인 (닫기)"** 버튼 클릭 → 이후 전체 토큰 값은 보이지 않음

> **제한**: 사용자당 최대 **10개**의 활성 토큰을 발급할 수 있습니다.

### 3.2. 토큰 목록 확인

발급된 토큰 목록이 표로 표시됩니다.

| 컬럼            | 설명                                         |
| --------------- | -------------------------------------------- |
| **이름**        | 발급 시 입력한 토큰 이름                     |
| **키 (마스킹)** | 토큰 앞 8자리만 표시 (`abc12345...(hidden)`) |
| **상태**        | 활성 / 비활성 / 만료됨                       |
| **만료일**      | 발급일로부터 1년 후 자동 만료                |
| **생성일**      | 토큰 발급일                                  |
| **관리**        | 삭제(비활성화) 버튼                          |

### 3.3. 토큰 삭제 (비활성화)

1. 삭제할 토큰 행의 **휴지통 아이콘** 클릭
2. 확인 다이얼로그에서 **"삭제"** 클릭
3. 토큰이 즉시 비활성화됨 → 해당 토큰을 사용하는 외부 연동이 즉시 차단됨

> 삭제된 토큰은 복구할 수 없습니다. 새 토큰을 발급하고 외부 시스템의 토큰 값을 교체해야 합니다.

---

## 4. 토큰 활용 예시

### 4.1. Jira Forge 앱 연동

발급한 토큰을 Forge 앱의 설정 화면(`Settings` 탭)에 입력합니다:

```
서버 URL: https://your-testcasecraft-server.com
API Token: [발급된 토큰 전체 값]
```

### 4.2. API 직접 호출 (curl 예시)

```bash
curl -H "X-API-Key: [발급된 토큰]" \
     https://your-server.com/api/jira/redirect/executions/PROJ-123
```

---

## 5. 보안 유의 사항

- 토큰은 **비밀번호와 동일하게** 취급하세요. 타인에게 공유하지 마세요.
- 더 이상 사용하지 않는 토큰은 즉시 **삭제(비활성화)** 하세요.
- 토큰이 유출되었다고 의심될 경우 즉시 삭제하고 새 토큰을 발급하세요.
- 토큰은 발급일로부터 **1년 후 자동 만료**됩니다. 만료 전 갱신이 필요합니다.

## 6. 관리자 전용 기능 (ADMIN 권한)

시스템 관리자는 REST API를 통해 모든 서비스 API 키를 직접 관리할 수 있습니다.

| 엔드포인트                                         | 메서드 | 설명                  |
| -------------------------------------------------- | ------ | --------------------- |
| `/api/admin/service-api-keys`                      | GET    | 전체 API 키 목록 조회 |
| `/api/admin/service-api-keys/generate?name={이름}` | POST   | API 키 발급           |
| `/api/admin/service-api-keys/{id}`                 | DELETE | API 키 비활성화       |

```bash
# 관리자 키 발급 예시
curl -X POST \
     -H "Authorization: Bearer {JWT 토큰}" \
     "http://localhost:8080/api/admin/service-api-keys/generate?name=JiraIntegrationKey"
```
