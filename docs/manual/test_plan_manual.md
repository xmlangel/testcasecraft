# 테스트 플랜 매뉴얼

## 1. 테스트 플랜 개요

테스트 플랜은 애플리케이션 내에서 테스트 케이스들을 체계적으로 그룹화하고 관리하기 위한 핵심 도구입니다. 특정 기능, 모듈 또는 릴리스에 대한 테스트 활동을 계획하고 실행하는 데 사용됩니다. 테스트 플랜을 통해 사용자는 관련된 테스트 케이스들을 한데 모아 효율적인 테스트 실행 및 결과 추적을 할 수 있습니다.

### 주요 목적:

- **테스트 실행 계획**: 특정 테스트 주기 또는 목표에 맞춰 실행할 테스트 케이스들을 정의합니다.
- **진행 상황 추적**: 테스트 플랜에 포함된 테스트 케이스들의 실행 진행 상황을 한눈에 파악할 수 있습니다.
- **결과 분석**: 테스트 플랜 단위로 테스트 결과를 집계하고 분석하여 품질 상태를 평가합니다.

## 2. 테스트 플랜 생성 및 관리

### 2.1. 테스트 플랜 생성

1.  **'테스트 플랜' 메뉴 접근**: 애플리케이션의 좌측 또는 상단 내비게이션에서 '테스트 플랜' 메뉴를 클릭합니다.
2.  **'새 테스트 플랜' 버튼 클릭**: 테스트 플랜 목록 화면에서 '새 테스트 플랜 생성' 또는 유사한 버튼을 클릭합니다.
3.  **정보 입력**: 다음 정보를 입력하여 테스트 플랜을 생성합니다.
    - **플랜 이름**: 테스트 플랜의 목적을 명확히 나타내는 이름 (예: "회원가입 기능 테스트", "v1.0 릴리스 회귀 테스트").
    - **설명**: 테스트 플랜에 대한 상세 설명 (선택 사항).
    - **프로젝트**: 이 테스트 플랜이 속할 프로젝트를 선택합니다.

### 2.2. 테스트 케이스 추가

생성된 테스트 플랜에 테스트 케이스를 추가하는 방법은 다음과 같습니다.

1.  **테스트 플랜 선택**: 목록에서 테스트 케이스를 추가할 테스트 플랜을 선택합니다.
2.  **'테스트 케이스 추가' 버튼 클릭**: 테스트 플랜 상세 화면에서 '테스트 케이스 추가' 버튼을 클릭합니다.
3.  **테스트 케이스 선택**: 프로젝트 내의 테스트 케이스 트리 또는 목록에서 추가하고자 하는 테스트 케이스들을 선택합니다. 여러 개의 테스트 케이스를 동시에 선택할 수 있습니다.
4.  **추가 완료**: 선택한 테스트 케이스들을 테스트 플랜에 추가합니다.

### 2.3. 테스트 플랜 수정 및 삭제

- **수정**: 테스트 플랜 상세 화면에서 이름, 설명 등을 수정할 수 있습니다.
- **삭제**: 테스트 플랜 목록 또는 상세 화면에서 해당 테스트 플랜을 삭제할 수 있습니다. (삭제 시 포함된 테스트 케이스는 삭제되지 않고, 플랜과의 연결만 해제됩니다.)

## 3. 테스트 플랜 실행

테스트 플랜은 테스트 실행의 기본 단위로 사용됩니다. 테스트 플랜에 포함된 테스트 케이스들은 일괄적으로 실행되거나 개별적으로 실행될 수 있습니다.

1.  **테스트 플랜 선택**: 실행하고자 하는 테스트 플랜을 선택합니다.
2.  **'테스트 실행' 시작**: 테스트 플랜 상세 화면에서 '테스트 실행' 버튼을 클릭합니다.
3.  **결과 기록**: 각 테스트 케이스의 실행 결과를 (성공, 실패, 블록됨 등) 기록하고, 필요한 경우 코멘트나 첨부 파일을 추가합니다.
4.  **진행 상황 확인**: 테스트 플랜의 전체 진행 상황과 각 테스트 케이스의 실행 상태를 실시간으로 확인할 수 있습니다.

## 4. 주요 기능

- **테스트 케이스 그룹화**: 관련된 테스트 케이스들을 논리적으로 묶어 관리합니다.
- **실행 주기 관리**: 특정 릴리스 또는 스프린트에 대한 테스트 범위를 정의합니다.
- **결과 집계**: 테스트 플랜 단위로 테스트 실행 결과를 집계하여 보고서를 생성합니다.
- **재사용성**: 한 번 생성된 테스트 플랜은 필요에 따라 여러 번 실행될 수 있습니다.
