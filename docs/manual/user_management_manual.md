# 사용자 관리 페이지 사용 매뉴얼

> **대상:** 시스템 관리자 (`role=ADMIN`)
> **메뉴 위치:** 헤더 **[관리 메뉴 ▾] → 사용자 관리** (`/users`)
> **권한:** ADMIN 외 사용자에게는 메뉴 자체가 노출되지 않음

## 1. 개요

이 문서는 testcasecraft의 **사용자 관리 페이지**(`/users`) 사용법을 안내합니다. 시스템 관리자가 사용자 계정을 효율적으로 조회·필터·수정·활성화/비활성화할 수 있는 종합 인터페이스를 제공합니다.

본인 프로필(비밀번호·언어·JIRA·API 토큰·테마)을 관리하는 화면은 별도이며, [§5 사용자 본인 프로필 관리](#5-사용자-본인-프로필-관리) 또는 통합 사용자 매뉴얼([`new/USER_MANUAL.md`](new/USER_MANUAL.md)) §13을 참조하세요.

## 2. 주요 기능 및 구성 요소

### 2.1. 사용자 통계 요약

페이지 상단에는 시스템의 사용자 현황을 한눈에 파악할 수 있는 4개의 통계 카드가 표시됩니다.

| 카드 | 의미 |
|---|---|
| **전체 사용자** | 시스템에 등록된 총 사용자 수 |
| **활성 사용자** | `isActive=true` 인 사용자 수 |
| **비활성 사용자** | `isActive=false` 인 사용자 수 |
| **최근 가입** | 최근 일정 기간 내 가입한 사용자 수 |

### 2.2. 사용자 목록 툴바 (검색, 필터, 작업)

사용자 목록 상단 툴바에서 사용자를 검색·필터링·일괄 작업을 수행합니다.

- **검색 필드** — **이름 · 사용자명 · 이메일** 3개 필드를 한 번에 키워드 검색 (백엔드 LIKE 검색)
- **역할 필터** — 드롭다운에서 다음 4가지 역할 또는 전체를 선택
  - **ADMIN** (시스템 관리자) · **MANAGER** (매니저) · **TESTER** (테스터) · **USER** (일반 사용자)
  - 각 역할은 고유 색상 칩으로 표시 (빨강·주황·파랑·초록)
- **상태 필터** — 전체 / 활성 / 비활성
- **🔄 새로고침** — 현재 검색 조건 그대로 목록을 다시 불러옵니다
- **⬇ 데이터 내보내기** — 현재 조회된 사용자 목록을 파일로 내보냅니다
- **[초기화]** — 모든 검색 키워드·필터·정렬을 기본값으로 리셋

### 2.3. 사용자 목록 테이블

| 컬럼 | 설명 |
|---|---|
| **사용자명** (`username`) | 로그인 ID. 정렬 가능 |
| **이름** (`name`) | 화면 표시명. 정렬 가능 |
| **이메일** (`email`) | 사용자 이메일 |
| **이메일 인증** | `미인증` / `인증` 칩 |
| **역할** | 4종 역할 색상 칩 + 아이콘 |
| **상태** | `활성` / `비활성` 색상 칩 |
| **가입일** (`createdAt`) | 등록일. 정렬 가능 |
| **최종 로그인** (`lastLoginAt`) | 마지막 로그인 시각. 정렬 가능. `없음` 표시 가능 |
| **작업** | 상세 보기 아이콘 + 점 3개 액션 메뉴 |

> 정렬은 **사용자명 · 이름 · 가입일 · 최종 로그인** 4개 컬럼에서 가능. 컬럼 헤더 클릭 → 오름차순 ↔ 내림차순 토글.

### 2.4. 페이지네이션

- **페이지 이동** — 이전/다음 또는 페이지 번호 클릭
- **페이지당 행 수** — `10 / 20 / 50 / 100` 중 선택 (기본 20)

### 2.5. 사용자 작업 (상세 보기 + 액션 메뉴)

각 사용자 행의 작업 열에는 두 가지 진입점이 있습니다.

#### 👁 상세 보기 (눈 아이콘)
사용자 상세 정보 다이얼로그를 엽니다. ([§2.7](#27-사용자-상세-정보-다이얼로그) 참조)

#### ⋮ 더 많은 작업 (점 3개 아이콘)
드롭다운 메뉴가 열리며 다음 항목이 노출됩니다.

| 항목 | 동작 | 노출 조건 |
|---|---|---|
| **상세 보기** | 상세 다이얼로그 열기 | 항상 |
| **활성화 / 비활성화** | 계정 상태 토글 (비활성 시 로그인 차단) | 항상 |
| **인증 이메일 발송** | 인증 메일을 다시 발송 | `emailVerified = false` 인 사용자만 |

### 2.6. 로딩 및 오류 처리

- **로딩 표시** — 목록을 불러오는 동안 로딩 스피너 표시
- **오류 메시지** — 데이터 로딩 실패 시 오류 메시지 + 재시도 옵션
- **검색 결과 없음** — 조건에 맞는 사용자가 없을 때 안내 메시지
- **스낵바 알림** — 활성화/비활성화·인증 이메일 발송 결과를 화면 하단 스낵바로 표시 (6초 자동 닫힘)

### 2.7. 사용자 상세 정보 다이얼로그

목록에서 **상세 보기**를 클릭하면 상세 다이얼로그가 열립니다.

#### 헤더
- 아바타 (역할별 아이콘) + 이름 + 사용자명(`@username`)
- **✏ 편집** 버튼 — 클릭 시 편집 모드. 편집 모드에서는 **💾 저장** + **❌ 취소** 노출
- **🔑 비밀번호 변경** 버튼 — 관리자가 해당 사용자의 비밀번호를 직접 재설정 ([§2.7.1](#271-관리자-비밀번호-변경))

#### 기본 정보 (편집 가능)
| 항목 | 편집 가능 | 비고 |
|---|---|---|
| **사용자명** | ❌ 변경 불가 | 회원가입 시점에 확정 |
| **이름** | ✅ | 필수, 공백 불가 |
| **이메일** | ✅ | 필수, 이메일 형식 검증 (`...@....`) |
| **역할** | ✅ | 드롭다운 — ADMIN/MANAGER/TESTER/USER 중 선택. 각 역할의 짧은 설명이 함께 표시됨 |
| **계정 활성화** | ✅ | Switch. OFF면 사용자는 로그인 불가 |

저장 시 이름/이메일 필수 검증 + 이메일 형식 검증 후 백엔드 호출. 저장 중에는 스피너 표시, 실패 시 다이얼로그 내 경고 메시지.

#### 상태 정보 (읽기 전용)
- **역할 아이콘 + 라벨** — 4종 역할의 시각적 표시
- **계정 상태** — `활성` / `비활성` 색상 칩 (active=success, inactive=outlined error)
- **이메일 인증 상태** — `인증 완료` / `미인증` 칩
- **활동 상태** — 최종 로그인 기준 환산 라벨: `최근 활동` / `일주일 내 활동` / `한 달 내 활동` / `장기 미접속`

#### 시간 정보
| 항목 | 의미 |
|---|---|
| **가입일** | 시스템 등록 시각 (`createdAt`) |
| **최종 수정일** | 사용자 정보 마지막 수정 시각 (`updatedAt`) |
| **최종 로그인** | 가장 최근 로그인 시각 (`lastLoginAt`) |
| **미접속 일수** | 최종 로그인 이후 경과일 |

#### 로딩 / 오류 / 편집 확인
- 정보 로딩 중에는 스피너 표시
- 로딩 실패 시 오류 메시지
- 편집 중 다이얼로그를 닫으려 하면 **저장하지 않고 닫으시겠습니까?** 확인 다이얼로그
- 저장 실패 시 다이얼로그 내 인라인 경고

#### 2.7.1. 관리자 비밀번호 변경

다이얼로그 상단의 **🔑 비밀번호 변경** 버튼을 클릭하면 별도 다이얼로그가 열립니다 (`AdminPasswordChangeDialog`).

- 관리자가 **현재 비밀번호 입력 없이** 사용자의 새 비밀번호를 직접 설정 가능
- 사용자가 비밀번호를 잊었거나 보안 사고로 즉시 변경이 필요한 경우 사용
- 변경 후 해당 사용자는 다음 로그인부터 새 비밀번호 사용

> 일반 사용자(본인)가 직접 자신의 비밀번호를 변경하는 화면은 **프로필 → 비밀번호** 탭이며, 이때는 현재 비밀번호 확인이 필요합니다.

## 3. 사용자 관리 페이지 사용 방법

1. **페이지 접근** — 헤더의 **[관리 메뉴 ▾] → 사용자 관리** 클릭
2. **검색 / 필터** — 키워드, 역할, 상태 필터 조합으로 원하는 사용자 추출
3. **정렬** — 컬럼 헤더 클릭으로 오름/내림차순 토글
4. **목록 탐색** — 페이지네이션 또는 페이지당 행 수 변경
5. **빠른 액션** — 점 3개 메뉴로 활성화/비활성화 또는 인증 이메일 재발송
6. **상세 정보 확인 및 편집**
   - 행의 👁 아이콘 또는 메뉴의 **상세 보기** 클릭
   - 정보 확인 후 ✏ **편집** 클릭
   - 이름·이메일·역할·계정 활성화 수정
   - 💾 **저장** 클릭하여 적용 (검증 실패 시 인라인 경고)
   - 취소하려면 ❌ **취소** 또는 다이얼로그 닫기 → 확인 후 닫힘
7. **비밀번호 직접 재설정** — 상세 다이얼로그의 🔑 **비밀번호 변경** 버튼 사용

## 4. 참고 사항

- **이메일 인증** — 회원가입 직후 사용자는 `미인증` 상태입니다. 인증을 완료하지 않으면 비밀번호 재설정 등 일부 기능이 제한됩니다. 관리자는 액션 메뉴로 인증 이메일을 재발송할 수 있습니다.
- **`USER` 역할 (일반 사용자)** — 회원가입한 사용자의 기본 역할입니다. 시스템 전역 권한이 가장 낮으며, 프로젝트별 권한은 별도(프로젝트 멤버 초대 시 부여).
- **데이터 내보내기** — 현재 조회된 결과(검색·필터 적용된) 목록을 파일로 다운로드. 감사·분석·백업에 활용.
- **활성화/비활성화 효과** — 비활성 계정은 로그인 불가. 기존 데이터(케이스·실행 결과·이동 기록 등)는 모두 보존되며, 재활성화 시 그대로 복귀.

### 4.1. 사용자 역할(Role) 4종 상세

| 코드 | 색상 | 권한 요약 |
|---|---|---|
| **ADMIN** | 빨강 (`#f44336`) | 시스템 전역 — 조직/사용자/메일/LLM/스케줄러/번역 설정 가능. 사용자 관리 페이지에 접근 가능 |
| **MANAGER** | 주황 (`#ff9800`) | 조직·프로젝트 관리에 준하는 중간 권한 |
| **TESTER** | 파랑 (`#2196f3`) | 케이스 실행 결과 기록·탐색 세션 진행 등 테스트 활동에 특화 |
| **USER** | 초록 (`#4caf50`) | 회원가입 기본값. 시스템 전역 권한 최소, 프로젝트별 권한은 별도 부여 |

> **시스템 전역 역할**과 **프로젝트별 역할(PROJECT_MANAGER, LEAD_DEVELOPER, TESTCASE_EDITOR, VIEWER)** 은 별개입니다. 동일 사용자가 시스템에서는 `USER`지만 특정 프로젝트의 PM일 수 있습니다.

## 5. 사용자 본인 프로필 관리

사용자는 우측 상단 **사용자 아바타 → 프로필** 메뉴로 본인 정보를 직접 관리합니다.

- **기본 정보** — 이름·이메일 수정, 이메일 인증 상태 확인 및 재발송
- **비밀번호 변경** — 현재 비밀번호 확인 후 새 비밀번호 설정
- **언어 설정** — 인터페이스 언어(한국어/English 등) 및 시간대(UTC/KST 등)
- **JIRA 설정** — 개인 JIRA 서버·이메일·API 토큰
- **Google Sheets 설정** — Google 계정 연결(OAuth)
- **API 토큰** — 외부 시스템 연동용 서비스 API 토큰 발급/회수 (한 사람당 최대 10개, 발급 직후 한 번만 전체 토큰 표시)
- **테마 설정** — 라이트/다크 모드

> API 토큰은 모든 역할(ADMIN/MANAGER/TESTER/USER)이 본인 계정에서 발급할 수 있습니다.
> 통합 사용자 매뉴얼: [`new/USER_MANUAL.md §13`](new/USER_MANUAL.md#13-사용자-프로필--설정)
> API 토큰 상세: [`api_token_manual.md`](./api_token_manual.md)
