# 테스트 결과 통계 및 분석 매뉴얼

## 1. 개요

테스트 결과 통계 및 분석 기능은 프로젝트의 전반적인 품질 상태를 입체적으로 분석하고 시각화하여 제공합니다. 단순한 패스/실패 여부를 넘어, 폴더 구조별, 테스트 플랜별, 실행자별로 성과를 비교하고 문제의 패턴을 식별하는 데 도움을 줍니다.

## 2. 주요 기능 및 구성 요소

### 2.1. 결과 요약 카드

대시보드 상단에는 현재 선택된 필터 조건에 따른 핵심 지표가 요약되어 표시됩니다.

- **총 테스트(Total)**: 분석 대상이 되는 전체 테스트 케이스 수.
- **성공률(Success Rate)**: 실행된 테스트 중 성공한 테스트의 비율.
- **실행률(Execution Rate)**: 전체 테스트 중 결과가 입력된 테스트의 비율.
- **JIRA 연동률**: 결함 관리를 위해 JIRA 이슈가 연결된 테스트의 비율.

### 2.2. 조회 필터 및 제어 (Statistics Filter Panel)

상황에 맞는 정밀한 데이터 분석을 위해 다양한 필터를 제공합니다.

- **테스트 플랜/실행 선택**: 특정 플랜이나 특정 차수의 실행 결과만 타겟팅하여 분석할 수 있습니다.
- **기간(Date Range)**: 최근 7일, 30일, 90일 또는 전체 기간을 설정합니다.
- **데이터 소스(Source)**: '수동 테스트', '자동화 테스트(JUnit)', 또는 두 결과를 합산한 '전체 합계' 중 선택할 수 있습니다.
- **새로고침**: 실시간으로 입력되는 최신 결과를 반영하기 위해 캐시를 삭제하고 데이터를 다시 불러옵니다.

## 3. 고급 분석 모드 (View Type)

'보기 형태' 필터를 통해 데이터 집계 기준을 변경할 수 있습니다.

### 3.1. 개요 (Overview)

프로젝트 전체의 품질 현황을 파악하기 위한 기본 뷰입니다. 성공, 실패, 블록됨, 건너뜀, 미실행 상태의 분포를 **파이 차트**로 시각화합니다.

### 3.2. 폴더별 분석 (By Folder) - 계층적 분석

프로젝트의 폴더 구조를 기반으로 품질을 분석하는 강력한 도구입니다.

- **계층 구조 반영**: 각 폴더의 통계에는 해당 폴더 하위에 포함된 모든 서브폴더와 테스트 케이스의 결과가 자동으로 합산됩니다.
- **Depth 필터**: `1`부터 `20`까지 깊이를 설정할 수 있습니다. 낮은 깊이를 선택하면 프로젝트의 큰 모듈별 상태를, 높은 깊이를 선택하면 상세 기능별 상태를 확인할 수 있습니다.
- **상세 통계 테이블**: 각 폴더별로 성공, 실패 등의 수치와 성공률(%)을 데이터 그리드로 제공합니다.

### 3.3. 실행자별/플랜별 비교 (By Executor / By Plan)

- **성과 비교**: 실행자별 또는 플랜별 결과를 **막대 차트**로 보여줍니다.
- **백분율 보기**: 차트에서 절대 수치 대신 비율(%)로 전환하여 상대적인 성과를 쉽게 비교할 수 있는 기능을 제공합니다.

## 4. 분석 결과 공유 및 보존 (URL 동기화)

시스템의 모든 필터 설정(조회 관점, Depth, 선택된 플랜 등)은 실시간으로 **URL 파라미터**에 반영됩니다.

- **공유**: 현재 보고 있는 분석 화면 그대로 URL을 복사하여 팀원에게 공유할 수 있습니다.
- **새로고침 보존**: 브라우저를 새로고침하더라도 기존에 설정했던 복잡한 필터 조건이 그대로 유지됩니다.

## 5. 활용 팁

- **품질 병목 구간 식별**: '폴더별 분석'에서 특정 모듈의 성공률이 비정상적으로 낮다면, 해당 기능에 대한 추가적인 품질 관리가 필요함을 의미합니다.
- **자동화 수준 파악**: '데이터 소스' 필터를 전환하며 수동 테스트와 자동화 테스트의 커버리지 및 성공률 차이를 비교해 보세요.
- **보고서 작성**: 통계 요약 정보와 시각화된 차트를 활용하여 주간/월간 품질 보고서를 손쉽게 작성할 수 있습니다.
