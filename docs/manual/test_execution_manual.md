# 테스트 실행 매뉴얼

## 1. 테스트 실행 개요

테스트 실행은 정의된 테스트 케이스 또는 테스트 플랜에 따라 실제 소프트웨어의 동작을 검증하고 그 결과를 기록하는 과정입니다. 이 과정을 통해 소프트웨어의 품질을 평가하고, 결함을 발견하며, 요구사항 충족 여부를 확인합니다. 효율적인 테스트 실행은 제품의 안정성과 신뢰성을 보장하는 데 필수적입니다.

### 주요 목적:

- **소프트웨어 품질 검증**: 기능이 올바르게 작동하는지 확인합니다.
- **결함 발견 및 추적**: 소프트웨어 내의 버그나 오류를 식별하고 관리합니다.
- **진행 상황 보고**: 테스트 활동의 현재 상태와 완료율을 파악합니다.
- **릴리스 결정 지원**: 테스트 결과를 바탕으로 제품 출시 여부를 결정하는 데 필요한 데이터를 제공합니다.

## 2. 테스트 실행 시작

테스트 실행은 주로 테스트 플랜을 통해 시작하거나, 개별 테스트 케이스 단위로 시작할 수 있습니다.

### 2.1. 테스트 플랜을 통한 실행 (권장)

1.  **테스트 플랜 선택**: '테스트 플랜' 메뉴에서 실행하고자 하는 테스트 플랜을 선택합니다.
2.  **'실행 시작' 버튼 클릭**: 테스트 플랜 상세 화면에서 '실행 시작' 또는 유사한 버튼을 클릭합니다. 이 버튼은 해당 플랜에 포함된 모든 테스트 케이스의 실행을 위한 인터페이스를 제공합니다.
3.  **실행 환경 설정 (선택 사항)**: 테스트 실행 환경(예: OS, 브라우저 버전)을 기록할 수 있습니다.

### 2.2. 개별 테스트 케이스 실행

1.  **테스트 케이스 선택**: '테스트 케이스' 메뉴에서 실행하고자 하는 개별 테스트 케이스를 선택합니다.
2.  **'실행' 버튼 클릭**: 테스트 케이스 상세 화면에서 '실행' 버튼을 클릭하여 해당 테스트 케이스의 실행을 시작합니다.

## 3. 테스트 결과 기록

테스트를 실행한 후에는 각 테스트 케이스의 결과를 정확하게 기록해야 합니다.

### 3.1. 결과 상태 선택

각 테스트 케이스에 대해 다음 중 하나의 상태를 선택합니다.

- **성공 (Pass)**: 테스트 케이스의 모든 단계가 예상대로 동작했습니다.
- **실패 (Fail)**: 테스트 케이스의 하나 이상의 단계가 예상과 다르게 동작했습니다. (결함 보고 필요)
- **블록됨 (Blocked)**: 다른 결함이나 환경 문제로 인해 테스트 케이스를 실행할 수 없습니다.
- **건너뜀 (Skipped)**: 특정 이유로 인해 테스트 케이스를 실행하지 않았습니다.

### 3.2. 상세 정보 기록

- **코멘트**: 테스트 실행 중 발생한 특이사항, 관찰 내용, 실패 원인 등을 상세히 기록합니다.
- **첨부 파일**: 스크린샷, 로그 파일, 동영상 등 테스트 결과를 증명하거나 결함 보고에 도움이 되는 자료를 첨부합니다.
- **결함 연결**: 실패한 테스트 케이스의 경우, 관련 결함(버그) 이슈를 생성하거나 기존 이슈에 연결합니다.

## 4. 실행 진행 상황 및 결과 확인

### 4.1. 진행 상황 모니터링

- **테스트 플랜 대시보드**: 테스트 플랜별로 전체 진행 상황(완료율, 성공/실패 비율)을 실시간으로 확인할 수 있습니다.
- **테스트 실행 이력**: 각 테스트 케이스의 과거 실행 이력을 확인하여 변경 사항이나 반복적인 실패 패턴을 분석할 수 있습니다.

### 4.2. 결과 보고서 생성

- **실행 보고서**: 테스트 플랜 또는 특정 기간 동안의 테스트 실행 결과를 요약한 보고서를 생성합니다. 이 보고서는 품질 평가 및 의사 결정에 활용됩니다.
- **결함 보고서**: 발견된 결함 목록과 그 상태를 추적하는 보고서를 생성합니다.

## 5. 주요 기능

- **유연한 실행 시작**: 테스트 플랜 또는 개별 테스트 케이스 단위로 실행을 시작할 수 있습니다.
- **다양한 결과 기록**: 성공, 실패, 블록됨 등 다양한 실행 상태를 기록하고 상세 코멘트 및 첨부 파일을 추가할 수 있습니다.
- **결함 관리 연동**: 발견된 결함을 즉시 결함 관리 시스템에 연결하여 추적성을 높입니다.
- **실시간 진행 상황**: 테스트 실행의 현재 상태를 실시간으로 모니터링하고 보고서를 생성합니다.
- **이력 관리**: 모든 테스트 실행 이력을 보관하여 추후 분석 및 회귀 테스트에 활용합니다.
