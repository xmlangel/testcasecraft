# 프로젝트 관리 페이지 사용 매뉴얼

## 1. 개요

이 문서는 테스트케이스 관리 시스템의 프로젝트 관리 페이지 사용 방법을 안내합니다. 프로젝트 관리 페이지는 사용자가 시스템 내의 모든 프로젝트를 조회, 생성, 수정, 이전 및 삭제할 수 있는 중앙 집중식 인터페이스를 제공합니다.

## 2. 주요 기능 및 구성 요소

### 2.1. 프로젝트 목록 및 분류

프로젝트 관리 페이지는 프로젝트를 효율적으로 탐색하고 관리할 수 있도록 여러 탭을 제공합니다.

*   **조직별 프로젝트 탭**: 사용자가 속한 조직별로 프로젝트를 분류하여 보여줍니다. 각 조직 아래에 해당 조직에 속한 프로젝트 목록이 표시됩니다.
*   **독립 프로젝트 탭**: 특정 조직에 속하지 않는 독립적인 프로젝트들을 보여줍니다. 개인 프로젝트나 조직 외부에서 관리되는 프로젝트에 적합합니다.
*   **전체 프로젝트 탭**: 사용자가 접근할 수 있는 모든 프로젝트를 조직 구분 없이 한눈에 보여줍니다.

각 프로젝트는 카드 형태로 표시되며, 프로젝트 이름, 코드, 설명, 소속 조직(또는 독립 프로젝트 여부), 테스트케이스 수, 멤버 수 등의 정보를 포함합니다.

### 2.2. 프로젝트 생성

*   **새 프로젝트 생성 버튼**: 페이지 상단 또는 각 조직 섹션 내에 위치한 "새 프로젝트 생성" 버튼을 클릭하여 새로운 프로젝트를 생성할 수 있습니다.
*   **프로젝트 생성 다이얼로그**: 프로젝트 이름, 코드, 설명, 그리고 소속될 조직을 선택할 수 있는 입력 필드를 제공합니다. 조직을 선택하지 않으면 독립 프로젝트로 생성됩니다.

### 2.3. 프로젝트 관리 작업 (프로젝트 카드 메뉴)

각 프로젝트 카드 우측 상단의 "더보기" (점 세 개) 아이콘을 클릭하면 해당 프로젝트에 대한 관리 메뉴가 나타납니다.

*   **프로젝트 열기**: 해당 프로젝트를 활성화하고, 프로젝트의 메인 대시보드 또는 기본 페이지로 이동합니다.
*   **수정**: 프로젝트 생성 다이얼로그와 유사한 형태로 프로젝트의 상세 정보를 수정할 수 있습니다.
*   **조직 이전**: 프로젝트를 다른 조직으로 옮기거나, 현재 조직에서 분리하여 독립 프로젝트로 전환할 수 있습니다.
*   **삭제**: 프로젝트를 영구적으로 삭제합니다. 이 작업은 되돌릴 수 없으며, 프로젝트에 속한 모든 테스트케이스 및 관련 데이터가 함께 삭제되므로 신중하게 사용해야 합니다. 삭제 전 확인 메시지가 표시됩니다.

### 2.4. 로딩 및 오류 처리

*   **로딩 표시**: 프로젝트 목록을 불러오거나 프로젝트 생성/수정/삭제와 같은 작업을 수행하는 동안 로딩 스피너가 표시됩니다.
*   **오류 메시지**: API 호출 실패나 입력 값 유효성 검사 오류 발생 시 사용자에게 알림 메시지가 표시됩니다.

## 3. 프로젝트 관리 페이지 사용 방법

1.  **페이지 접근**: 네비게이션 바에서 "프로젝트 선택" 버튼을 클릭하여 프로젝트 관리 페이지로 이동합니다.
2.  **프로젝트 조회**: 상단의 탭을 클릭하여 "조직별 프로젝트", "독립 프로젝트", ""전체 프로젝트"" 중 원하는 분류로 프로젝트 목록을 확인합니다.
3.  **새 프로젝트 생성**: "새 프로젝트 생성" 버튼을 클릭하고, 다이얼로그에 필요한 정보를 입력한 후 "생성" 버튼을 클릭합니다.
4.  **프로젝트 열기**: 원하는 프로젝트 카드 하단의 "프로젝트 열기" 버튼을 클릭하여 해당 프로젝트로 이동합니다.
5.  **프로젝트 수정/이전/삭제**: 프로젝트 카드 우측 상단의 "더보기" 아이콘을 클릭하여 메뉴를 열고, 원하는 작업을 선택하여 진행합니다. 각 작업에 대한 다이얼로그의 안내를 따릅니다.

## 4. 참고 사항

*   프로젝트 생성 시 프로젝트 이름과 코드는 필수 입력 항목입니다.
*   프로젝트 코드는 프로젝트를 식별하는 고유한 값으로 사용됩니다.
*   프로젝트 삭제는 영구적인 작업이므로 신중하게 결정해야 합니다.
