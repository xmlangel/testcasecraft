# 테스트케이스 관리 매뉴얼

## 1. 개요

이 문서는 프로젝트 내 테스트케이스를 효율적으로 관리하기 위한 방법을 안내합니다. 테스트케이스 관리는 테스트 폴더를 생성하여 테스트케이스를 계층적으로 구성하고, 각 테스트케이스의 상세 정보를 입력하며, 필요에 따라 일괄 편집하는 기능을 포함합니다.

## 2. 주요 기능 및 구성 요소

테스트케이스 관리는 크게 세 가지 주요 컴포넌트를 통해 이루어집니다.

### 2.1. 테스트케이스 트리 (TestCase Tree)

프로젝트의 모든 테스트 폴더와 테스트케이스를 계층적 구조로 보여주는 탐색기 형태의 인터페이스입니다.

- **계층적 보기**: 폴더와 테스트케이스를 트리 형태로 구성하여 직관적인 관리가 가능합니다.
- **폴더/테스트케이스 생성**: 특정 폴더 아래 또는 루트에 새로운 폴더나 테스트케이스를 추가할 수 있습니다.
- **이름 변경**: 폴더나 테스트케이스의 이름을 변경할 수 있습니다.
- **삭제**: 선택된 폴더나 테스트케이스를 삭제합니다. 폴더 삭제 시 하위의 모든 항목이 함께 삭제됩니다.
- **순서 변경**: 폴더나 테스트케이스의 표시 순서를 변경할 수 있습니다.
- **일괄 선택/삭제**: 체크박스를 통해 여러 테스트케이스를 선택하고 일괄 삭제할 수 있습니다.

### 2.2. 테스트케이스 입력 모드 (TestCase Hybrid Form)

테스트케이스의 상세 정보를 입력하거나 편집하는 방식을 선택할 수 있는 인터페이스입니다. 단일 폼 모드와 스프레드시트 모드 간 전환이 가능합니다.

- **폼 모드**: 개별 테스트케이스의 상세 정보를 입력하고 편집하는 데 사용됩니다.
- **스프레드시트 모드**: 여러 테스트케이스를 동시에 빠르고 효율적으로 편집하는 데 사용됩니다.

### 2.3. 테스트케이스 상세 폼 (TestCase Form)

단일 테스트케이스의 모든 상세 정보를 입력하고 관리하는 데 사용되는 폼입니다.

- **기본 정보**: 테스트케이스 이름, 설명, 사전 조건(Pre-condition)을 입력합니다.
- **테스트 스텝**: 각 스텝별로 설명(Step)과 예상 결과(Expected)를 테이블 형태로 입력합니다. 스텝 추가 및 삭제가 가능합니다.
- **전체 예상 결과**: 모든 스텝을 아우르는 최종 예상 결과를 입력합니다.

### 2.4. 테스트케이스 스프레드시트 (TestCase Spreadsheet)

여러 테스트케이스를 엑셀과 유사한 스프레드시트 형태로 일괄 편집할 수 있는 강력한 기능입니다.

- **일괄 편집**: 여러 테스트케이스의 이름, 설명, 사전 조건, 예상 결과 및 각 스텝의 내용을 한 화면에서 동시에 편집할 수 있습니다.
- **동적 스텝 관리**: 스텝의 개수를 유연하게 조절할 수 있으며, 최대 10개까지 스텝을 추가할 수 있습니다.
- **행 추가**: 새로운 테스트케이스를 추가하기 위해 빈 행을 추가할 수 있습니다.
- **일괄 저장**: 스프레드시트에서 변경된 모든 내용을 한 번에 저장합니다.
- **새로고침**: 백엔드에서 최신 테스트케이스 데이터를 가져와 스프레드시트를 업데이트합니다.

## 3. 테스트케이스 관리 사용 방법

### 3.1. 테스트케이스 선택 및 조회

1.  **트리에서 선택**: 좌측의 테스트케이스 트리에서 원하는 테스트케이스나 폴더를 클릭합니다.
2.  선택된 항목의 상세 정보가 우측의 폼 또는 스프레드시트에 표시됩니다.

### 3.2. 새 테스트케이스/폴더 생성

1.  **루트 또는 특정 폴더에 추가**: 트리 상단의 "+" 버튼을 클릭하거나, 특정 폴더를 우클릭하여 컨텍스트 메뉴를 엽니다.
2.  "폴더 추가" 또는 "테스트케이스 추가"를 선택합니다.
3.  새 항목의 이름을 입력하고 저장합니다.

### 3.3. 테스트케이스/폴더 이름 변경

1.  트리에서 이름을 변경할 항목을 우클릭하여 컨텍스트 메뉴를 엽니다.
2.  "이름 변경"을 선택하고 새 이름을 입력한 후 저장합니다.

### 3.4. 테스트케이스/폴더 삭제

1.  **개별 삭제**: 트리에서 삭제할 항목을 우클릭하여 컨텍스트 메뉴를 엽니다. "삭제"를 선택하고 확인 메시지에 동의합니다.
2.  **일괄 삭제**: 트리에서 삭제할 테스트케이스 옆의 체크박스를 선택한 후, 트리 상단의 "휴지통" 아이콘을 클릭하여 일괄 삭제합니다.
3.  **폼에서 삭제**(v1.0.93~): 케이스를 폼 모드로 열면 헤더 우측의 **[삭제]** 버튼으로도 삭제할 수 있습니다. 트리 삭제와 동일한 확인 창(삭제 대상 ID·이름 표시)이 떠서 일관된 방식으로 확정합니다.

### 3.5. 테스트케이스 상세 정보 편집 (폼 모드)

1.  우측 상단의 "폼" 모드를 선택합니다.
2.  테스트케이스 이름, 설명, 사전 조건, 전체 예상 결과를 입력하거나 수정합니다.
3.  "테스트 스텝" 섹션에서 "스텝 추가" 버튼을 클릭하여 새로운 스텝을 추가하거나, 기존 스텝의 설명과 예상 결과를 수정합니다. 스텝 옆의 "휴지통" 아이콘으로 스텝을 삭제할 수 있습니다.
4.  모든 변경 사항을 입력한 후 하단의 "저장" 버튼을 클릭합니다.

### 3.6. 테스트케이스 일괄 편집 (스프레드시트 모드)

1.  우측 상단의 "스프레드시트" 모드를 선택합니다.
2.  엑셀처럼 각 셀을 클릭하여 테스트케이스의 이름, 설명, 사전 조건, 예상 결과 및 각 스텝의 내용을 직접 편집합니다.
3.  "행 추가" 버튼을 클릭하여 새로운 테스트케이스를 위한 빈 행을 추가할 수 있습니다.
4.  "⚙️" 버튼을 클릭하여 스텝 수를 조정할 수 있습니다 (최대 10개).
5.  모든 변경 사항을 입력한 후 "일괄 저장" 버튼을 클릭하여 한 번에 저장합니다.
6.  "새로고침" 버튼을 클릭하여 백엔드에서 최신 데이터를 가져와 스프레드시트를 업데이트할 수 있습니다.

### 3.7. 다른 프로젝트로 이동·복사 (v1.0.93~)

같은 프로젝트 안에서의 위치 변경은 드래그앤드롭으로 하지만, 프로젝트를 넘나드는 이동·복사는 전용 버튼으로 수행합니다.

1.  트리에서 옮길 케이스·폴더의 **체크박스**로 하나 이상 선택합니다.
2.  트리 헤더의 **[프로젝트 이동/복사]** 버튼(선택 항목이 있을 때만 노출)을 클릭합니다.
3.  "테스트케이스 일괄 작업" 다이얼로그에서 작업 유형(**이동** 또는 **복사**)과 **대상 프로젝트**(필수)·**대상 폴더**(선택, 미지정 시 루트)를 고른 뒤 **[실행]** 합니다.
4.  **이동**은 연결된 테스트 결과를 함께 옮기며(대상 프로젝트에 대응 실행을 새로 생성) 출발·대상 모두 편집 권한이 필요합니다. **복사**는 케이스만 복제하며(결과 미포함) 출발 조회 + 대상 편집 권한이 필요합니다.

## 4. 참고 사항

- 폴더 삭제 시 하위의 모든 테스트케이스와 폴더가 함께 삭제되므로 주의해야 합니다.
- 스프레드시트 모드에서 변경된 내용은 "일괄 저장" 버튼을 클릭하기 전까지는 임시 상태이며, 저장하지 않고 페이지를 이동하면 손실될 수 있습니다.
- 역할에 따라 테스트케이스 생성, 수정, 삭제 권한이 제한될 수 있습니다.
