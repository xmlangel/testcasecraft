## 전체 구조 요약

`README_USAGE.md` 파일을 기반으로, 이 테스트케이스 관리 툴(React 기반)의 전체 구조는 아래와 같습니다[^1].

---

### **1. 주요 폴더 및 파일 구조**

- **public/**
    - `index.html`
    - `favicon.ico`
- **src/**
    - **components/**
        - `TestCaseTree.js` : 테스트케이스 트리 뷰(폴더/케이스 관리)
        - `TestCaseForm.js` : 테스트케이스 상세/수정/단계 관리
        - `TestPlanList.js` : 테스트플랜 목록
        - `TestPlanForm.js` : 테스트플랜 생성/수정
        - `TestExecutionList.js` : 테스트 실행 목록
        - `TestExecutionForm.js` : 테스트 실행 상세/수정
        - `TestResultForm.js` : 테스트 결과 입력/수정
    - **models/**
        - `testCase.js` : 테스트케이스/폴더/단계 모델 정의
        - `testPlan.js` : 테스트플랜 모델 정의
        - `testExecution.js` : 테스트 실행/결과/상태 모델 정의
    - **context/**
        - `AppContext.js` : 전역 상태관리(Context API, reducer, 액션 등)
    - **utils/**
        - `treeUtils.js` : 트리구조 변환, 탐색, 유틸 함수
    - `App.js` : 전체 앱 구조 및 탭/라우팅
    - `index.js` : 엔트리포인트
    - `styles.css` : 전역 스타일
- **기타**
    - `package.json` : 의존성, 스크립트, 빌드/실행 설정
    - `README.md` : 일반 설명

---

### **2. 주요 기능별 구조**

#### **A. 테스트케이스 관리**

- **트리 구조**로 폴더/테스트케이스 생성, 이름 변경, 삭제, 이동 지원
- 각 테스트케이스는 **단계(steps)**, **설명**, **예상결과** 등 속성 관리
- 트리뷰에서 우클릭 메뉴로 폴더/케이스 추가, 삭제, 이름변경 가능


#### **B. 테스트플랜 관리**

- 테스트플랜은 **여러 테스트케이스를 선택**해 구성
- 플랜 생성/수정 시, 트리에서 테스트케이스 선택(체크박스)
- 플랜 목록에서 편집/삭제/실행 시작 가능


#### **C. 테스트 실행 및 결과 관리**

- 테스트플랜을 기반으로 **테스트 실행(Execution)** 생성
- 실행 상태: 미시작, 진행중, 완료
- 각 실행별로 **테스트케이스별 결과(PASS/FAIL/BLOCKED/NOTRUN)**, 노트 입력
- 진행률(%) 표시, 결과별 아이콘/색상 표시
- 실행 상세에서 결과 직접 입력 및 실행 완료 처리 가능


#### **D. 상태관리 및 데이터 흐름**

- **Context API + useReducer**로 전역 상태 관리
- 주요 액션: 추가/수정/삭제/활성화/실행시작/완료/결과입력 등
- 상태는 localStorage에 저장, 새로고침 후에도 복원


#### **E. UI/UX**

- **Material UI** 기반의 일관된 디자인
- 트리뷰, 리스트, 폼, 다이얼로그, 탭 등 다양한 컴포넌트 활용
- 반응형 레이아웃(Grid, Paper 등)

---

### **3. 앱 실행 흐름**

1. **탭 구조**:
    - 테스트케이스 관리
    - 테스트플랜 관리
    - 테스트 실행 관리
2. **각 탭별로 주요 컴포넌트 배치**:
    - 트리뷰 + 상세폼, 플랜리스트 + 플랜폼, 실행리스트 + 실행폼 등
3. **CRUD 및 실행/결과 입력**:
    - 각 엔티티별로 추가, 수정, 삭제, 실행, 결과입력 등 전 기능 지원

---

### **4. 의존성 및 스택**

- **React 18**
- **Material UI, MUI Icons, MUI X TreeView**
- **emotion (스타일링)**
- **react-beautiful-dnd** (드래그앤드롭)
- **uuid** (ID 생성)
- **localStorage** (상태 저장)
