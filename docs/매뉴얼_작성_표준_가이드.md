# 매뉴얼(사용 설명서) 작성 표준 가이드

> 기술 문서·사용 설명서 작성에 적용되는 국제 표준(ISO / IEC / IEEE / ANSI)을 정리하고, 소프트웨어 제품 문서화에 어떻게 적용할지를 다룬 실무 가이드.

---

## 0. 한눈에 보기

| 분류                  | 표준                                | 용도                                                        |
| --------------------- | ----------------------------------- | ----------------------------------------------------------- |
| **최상위(공통) 표준** | **IEC/IEEE 82079-1:2019**           | 모든 제품의 "사용 정보(information for use)" 작성 일반 원칙 |
| **소프트웨어 문서**   | **ISO/IEC/IEEE 26511~26515, 26531** | 소프트웨어 사용자 문서 (역할별)                             |
| 산업: 기계            | ISO 20607                           | 기계류 사용 설명서                                          |
| 산업: 디지털 매뉴얼   | ISO 21600:2019                      | 기계 제품 디지털 매뉴얼                                     |
| 산업: 의료기기        | IFU 관련 규정                       | 의료기기 사용 지침                                          |
| 안전 정보             | ANSI Z535.6                         | 제품 안전 경고·기호                                         |
| 기술 도면/문서(TPD)   | ISO 128, ISO 7200                   | 도면 표현·표제란                                            |
| 품질경영(QMS) 문서    | ISO 9001 계열 작업지시서            | 사내 절차·작업지시                                          |

> 단일한 "전 세계 통합 매뉴얼 표준"은 없습니다. 대신 **공통 원칙 표준(82079-1)** 위에 **산업별·역할별 표준**이 얹히는 구조입니다.

## 1. 최상위 표준 — IEC/IEEE 82079-1:2019

가장 먼저 알아야 할 핵심 표준. 정식 명칭은
**"Preparation of information for use (instructions for use) of products — Part 1: Principles and general requirements"**.

### 개요

- IEC·IEEE·ISO가 **공동 개발**한 horizontal(수평·공통) 표준.
- 2019년 5월 2판(Ed.2) 발행, 기존 IEC 82079-1:2012을 대체.
- 적용 범위가 매우 넓음: **소비재 → 복잡한 산업 설비, 그리고 소프트웨어까지** 포함.
- 숙련자(전문가)와 비숙련자(일반 소비자) 모두를 대상 독자로 고려.

### 핵심 개념 — 정보 유형(Information Types)

82079-1은 사용 정보를 다음 3가지 유형으로 구조화하도록 권고합니다. **매뉴얼을 쓸 때 이 분류를 기준으로 챕터/섹션을 나누는 것이 핵심.**

1. **개념 정보 (Conceptual)** — 제품이 무엇이고 왜 그런지. 개요, 안전 주의사항, 시스템 설명.
2. **지침 정보 (Instructional)** — 어떻게 하는지. 단계별 절차(step-by-step), 경고 메시지.
3. **참조 정보 (Reference)** — 찾아보는 정보. 사양, 트러블슈팅, 용어집, 색인.

### 품질 기준

표준이 정의하는 사용 정보의 목적:

> "대상 독자가 제품을 **안전하고(safe), 효율적이며(efficient), 효과적으로(effective)** 사용할 수 있도록 필요한 정보를 제공해야 한다."

- 검색·탐색이 쉽도록 구조화할 것 (목차, 검색, 명확한 네비게이션).
- 대상 독자가 여러 그룹(설치자/운영자/유지보수자 등)이면 **챕터·섹션을 분리**할 것.
- 정보 작성자의 **전문 역량(competency)** 요구사항도 명시.
- **경험적 평가 방법**(사용성 테스트 등)을 정규(normative) 절차에 포함.

### 왜 중요한가

- 다른 제품별 표준들이 **참조하는 기반 표준**임. (예: 기계 → ISO 20607, 소프트웨어 → ISO/IEC/IEEE 26514가 82079-1 위에 구축됨.)
- 부실한 설명서는 **사용자 불만 + 법적 책임(liability)** 의 원천이 되므로, 표준 준수는 리스크 관리 차원에서도 의미가 있음.

## 2. 소프트웨어 문서 표준 — ISO/IEC/IEEE 2651x 시리즈 ⭐

**소프트웨어/플랫폼 제품(예: 그래프 DB, RAG 플랫폼)을 문서화한다면 이 시리즈가 가장 직접적으로 관련됩니다.** ISO·IEC·IEEE 합동 기술위원회(JTC1/SC7)가 개발한 소프트웨어 사용자 문서 표준 묶음으로, **역할별로 나뉜 것**이 특징입니다.

| 표준      | 대상 역할         | 내용                                                                          |
| --------- | ----------------- | ----------------------------------------------------------------------------- |
| **26511** | 문서화 **관리자** | 전체 라이프사이클의 문서 관리. 문서화 계획, 측정·추정, 변경관리, 팀 구성·역할 |
| **26512** | **발주자·공급자** | 문서화 서비스의 구매·제공 규칙. 클라이언트-벤더 협력                          |
| **26513** | **테스터·리뷰어** | 문서 평가·검증. 리뷰, 사용성 테스트, 현지화/커스터마이징 테스트               |
| **26514** | **설계자·개발자** | 사용자 문서의 **구조·내용·형식** + 스타일 가이드. (제일 실무적)               |
| **26515** | (애자일)          | **애자일 환경**에서의 사용자 문서 개발                                        |
| **26531** | 콘텐츠 관리자     | 콘텐츠 관리 프로세스                                                          |

### 알아둘 점

- **26514:2022**이 최신 개정판(설계·개발자용). 핵심: 문서를 일회성 "책 한 권"이 아니라, 지속적으로 관리되는 **정보 제품(information product)** 으로 보는 관점으로 전환.
- 최근 개정에서 **"documentation" → "information for users"** 로 용어를 통일. (인쇄물 중심 사고에서 벗어나기 위함.)
- 소프트웨어 툴에 **독립적**으로 작성됨 → 인쇄/온스크린 문서 모두 적용 가능.

### 실무 시사점 (Skai Worldwide 적용 관점)

- **26514** = 매뉴얼을 실제로 쓰는 사람의 표준 → 구조/형식 기준으로 채택.
- **26515** = 빠르게 변하는 제품(PG17/PG18 머지 등)을 따라가야 하면, 애자일 문서화 원칙이 유용.
- **26513** = 문서 리뷰/QA 프로세스 정립 시 참고. (지식베이스 감사에서 "postmortem culture 부재" 같은 갭을 메울 때 응용 가능.)

## 3. 산업별 표준

### ISO 20607 — 기계류 사용 설명서

- 기계류(machinery)에 **특화된 사용 설명서 표준**.
- ISO 12100(안전)과 IEC 82079-1을 기반으로 구축, 기계에 초점.
- EU 기계 지침(Machinery Directive 2006/42/EC) 준수에 활용. → **CE 마킹** 관련.

### ISO 21600:2019 — 기계 제품 **디지털** 매뉴얼

- 디지털 매뉴얼의 구성, 기본 원칙, 준비 프로세스, 발행·관리 요구사항을 규정.
- 사용·수리·유지보수 안내용 디지털 매뉴얼 작성 지침.
- 특정 소프트웨어의 기능/데이터 구조는 규정하지 않음 (플랫폼 중립).

### 의료기기 — IFU (Instructions For Use)

- 의료기기는 별도의 강한 규제 체계(IFU)가 적용됨. (해당 산업이면 별도 확인 필요.)

## 4. 안전 정보 — ANSI Z535.6

- 정식: _American National Standard for Product Safety Information in Product Manuals, Instructions, and Other Collateral Materials._
- **제품 매뉴얼 안에 안전 정보(경고·주의·위험 메시지)를 어떻게 배치·표기할지**에 대한 미국 표준.
- 안전 기호·경고 형식은 ANSI Z535 계열이 담당.
- 위험을 동반하는 제품 문서라면 82079-1과 함께 적용 권장.

---

## 5. 기술 제품 문서(TPD) 관련 표준

도면·표제란 등 기술 문서 형식 표준 (매뉴얼에 도면이 들어갈 때 참고):

- **ISO 128 시리즈** — 기술 제품 문서의 표현 일반 원칙 (선·뷰·단면 등). 2020·2022년 최신 개정.
- **ISO 7200** — 기술 도면 표제란(title block)의 데이터 필드 형식.

---

## 6. 표준은 아니지만 사실상 표준인 것 (업계 프레임워크)

ISO 표준은 아니지만 기술 문서 업계에서 널리 쓰이는 방법론:

- **DITA** (Darwin Information Typing Architecture) — 토픽 기반 구조화 문서 작성 표준(OASIS). 재사용·단일 소스 관리에 강력.
- **S1000D** — 항공·방산 분야 기술 출판 국제 사양.
- **Simplified Technical English (STE / ASD-STE100)** — 영어 기술 문서를 명확·일관되게 쓰기 위한 통제 언어.
- **Minimalism (정보 최소주의)** — 사용자가 실제 작업에 필요한 것만 담는 작성 원칙.
- **Markdown / reStructuredText + Docs-as-Code** — 개발 친화적 문서화. (Git 기반 워크플로우와 잘 맞음.)

## 7. 실무 적용 체크리스트

표준들에서 공통적으로 도출되는 **좋은 매뉴얼의 원칙**:

### 구조

- [ ] 정보를 **개념 / 지침 / 참조** 3유형으로 분리했는가?
- [ ] 대상 독자(설치자·운영자·관리자·개발자 등)별로 섹션을 나눴는가?
- [ ] 목차·색인·검색 등 **탐색 수단**이 있는가?
- [ ] **개정 이력(revision history)** 페이지가 있는가?

### 작성 (문장)

- [ ] 한 문장에 하나의 핵심 아이디어 — **짧고 명확하게**.
- [ ] 절차는 **단락이 아니라 번호/불릿 목록**으로.
- [ ] 누구나 아는 **쉬운 용어**, 통제된 어휘 사용.
- [ ] 한 문서에 너무 많은 정보를 욱여넣지 말 것. 동시에 같은 주제를 너무 잘게 쪼개지도 말 것.

### 시각 자료

- [ ] 표·차트·그래프·스크린샷을 적극 활용 ("그림 한 장이 천 마디").
- [ ] 경고/주의 정보는 ANSI Z535 형식 등 일관된 표기.

### 품질

- [ ] **안전·효율·효과** 3대 목적을 충족하는가?
- [ ] 실제 사용자로 **사용성 테스트(empirical evaluation)** 를 했는가?
- [ ] 리뷰/테스트 프로세스가 있는가? (→ ISO/IEC/IEEE 26513)

## 8. 권장 적용 전략 (소프트웨어/플랫폼 제품 기준)

1. **기반 원칙**: IEC/IEEE 82079-1:2019의 정보 유형 분류와 품질 기준을 전사 문서 원칙으로 채택.
2. **소프트웨어 문서 표준**: ISO/IEC/IEEE **26514**(구조·형식) + **26515**(애자일) 채택.
3. **문서 거버넌스**: 26511(관리)·26513(리뷰/테스트)을 참고해 문서 QA·개정 프로세스 수립.
4. **작성 방식**: DITA 또는 Docs-as-Code(Markdown + Git) 같은 단일 소스·토픽 기반 방식으로 재사용성 확보.
5. **통제 언어**: 다국어(한/영) 문서라면 STE 같은 통제 영어 + 번역 친화적 문장 구조 고려.
