# 테스트 케이스 트리 — 드래그앤드롭 사용 가이드

TestcaseCraft의 테스트 케이스 트리에서 폴더와 케이스를 **마우스로 끌어다 놓는 것만으로** 위치를 바꿀 수 있습니다. 기존의 우클릭 메뉴와 "순서 편집 모드"의 위/아래 버튼은 그대로 함께 사용할 수 있습니다.

## 1. 기본 동작

| 하고 싶은 것 | 동작 |
|---|---|
| 케이스를 다른 폴더로 옮기기 | 케이스를 끌어서 폴더 위에 놓기 |
| 폴더 통째로 옮기기 | 폴더 행을 끌어서 다른 폴더 위에 놓기 |
| 같은 부모 내 순서 바꾸기 | 노드 사이의 **가는 갭**에 놓기 (그 자리에 삽입됨) |
| 루트로 보내기 | 트리 최상단 빈 영역에 놓기 |

드롭 직전, 어디에 들어갈지 **연한 가이드 라인** 으로 미리 보입니다.

## 2. 다중 선택 이동

여러 개를 한 번에 옮길 수 있습니다.

- **Cmd-click (macOS) / Ctrl-click (Win/Linux)**: 개별 항목 토글 선택
- **Shift-click**: 마지막 선택 항목부터 현재 항목까지 범위 선택
- 선택된 묶음 중 하나를 드래그하면 **선택된 모든 항목** 이 함께 이동합니다
- 선택되지 않은 항목을 드래그하면 **그 항목 하나만** 이동합니다 (선택 묶음은 유지)

## 3. 거부되는 이동

다음은 자동으로 막힙니다. 드래그 중 가이드가 **빨간색**으로 변하고 커서가 `🚫` 표시로 바뀝니다.

- 자기 자신 위에 놓기
- 자기 **하위(후손)** 폴더에 놓기 (순환 차단)
- 다른 프로젝트 폴더에 놓기
- 케이스 **아래에** 다른 노드를 놓기 (케이스는 자식을 가질 수 없음)
- 시스템 기본 폴더를 이동 (`systemFolder` 타입은 보호됨)
- 시스템 기본 폴더를 다른 곳으로 옮기는 시도

거부 시 토스트로 사유를 알려드립니다.

## 4. 실패와 복원

- 네트워크/서버 오류로 이동이 실패하면 트리는 **자동으로 원래 상태로 되돌아갑니다**.
- 페이지를 새로고침해도 위치는 유지됩니다 (서버 저장 성공 시).

## 5. 감사 로그 (Audit Log)

모든 성공한 이동은 서버 `tc_move_audit_log` 테이블에 기록됩니다. 누가, 언제, 어디서 어디로 옮겼는지 추적 가능합니다. 관리자는 DB 또는 향후 제공될 관리 화면에서 조회할 수 있습니다.

## 6. 기존 방식과의 공존

DnD가 마음에 들지 않거나, 화면이 좁을 때는 **기존 방식**을 그대로 쓸 수 있습니다.

- 우클릭 → 폴더/케이스 추가, 이름 변경, 삭제, 버전 히스토리
- "순서 편집 모드"의 위/아래 버튼으로 같은 부모 내 순서 변경
- API (또는 MCP `testcase_move`)로 직접 이동

DnD는 이들 위에 **추가된 진입점** 일 뿐입니다.

## 7. 자주 묻는 질문

**Q. 드래그가 안 되는 노드가 있어요.**
A. `systemFolder` 타입(예: 시스템 기본 폴더)은 의도적으로 보호됩니다.

**Q. 여러 개 선택했는데 하나만 옮겨졌어요.**
A. 드래그 시작 직전에 클릭한 노드가 선택 묶음 **밖**이면 그 노드 한 개만 옮겨집니다. Cmd/Shift-click으로 선택을 먼저 만든 뒤, 선택된 항목 위에서 드래그를 시작하세요.

**Q. 매우 큰 폴더(자식 수백 개)로 옮기면 느리지 않나요?**
A. 현재 구현은 같은 부모 형제의 `displayOrder`를 1부터 정규화합니다. 자식이 수백 단위면 체감 가능합니다. 자식이 1000+ 인 폴더에서 성능 이슈가 보고되면 Phase 2에서 fractional indexing(lexorank)로 전환합니다.
