# Google Sheets 연동 가이드

이 문서는 TestcaseCraft에서 Google Sheets 내보내기 기능을 사용하기 위해 필요한 구글 서비스 계정 설정 방법을 안내합니다.

---

## 🚀 설정 단계 요약

Google Sheets 내보내기 기능을 사용하려면 **Google 서비스 계정(Service Account)**을 생성하고 발급받은 **JSON 키**를 애플리케이션에 등록해야 합니다.

### 1단계: Google Cloud 프로젝트 준비
1. [Google Cloud Console](https://console.cloud.google.com/)에 접속하여 로그인합니다.
2. 상단 바에서 프로젝트를 선택하거나 **[새 프로젝트]**를 생성합니다.

### 2단계: API 활성화
1. **[API 및 서비스] > [라이브러리]** 메뉴로 이동합니다.
2. **"Google Sheets API"**를 검색하여 **[사용]** 버튼을 클릭합니다.

### 3단계: 서비스 계정 및 키 생성
1. **[IAM 및 관리] > [서비스 계정]** 메뉴로 이동합니다.
2. 상단의 **[+ 서비스 계정 만들기]**를 클릭하여 계정을 생성합니다 (이름은 자유롭게 설정).
3. 생성된 계정의 **[관리]** 메뉴 또는 이메일 주소를 클릭하여 상세 화면으로 들어갑니다.
4. **[키]** 탭을 선택하고 **[키 추가] > [새 키 만들기]**를 클릭합니다.
5. 키 유형으로 **JSON**을 선택하고 만들기 버튼을 누릅니다. 파일이 자동으로 다운로드됩니다.

### 4단계: 애플리케이션에 등록
1. 다운로드받은 JSON 파일을 메모장 등으로 열어 **내용 전체**를 복사합니다.
2. 애플리케이션의 **[우측 상단 아바타 메뉴] > [Google Sheets 설정]** 페이지로 들어갑니다.
3. 복사한 JSON 내용을 입력란에 붙여넣고 **[연동 설정 저장]**을 클릭합니다.

### 5단계: 대상 구글 시트에 권한 부여 (필수)
> [!IMPORTANT]
> 이 단계를 건너뛰면 '권한 부족' 오류가 발생합니다.
1. 내보내기를 할 대상 **구글 시트 파일**을 엽니다.
2. 우측 상단의 **[공유]** 버튼을 클릭합니다.
3. 서비스 계정의 **이메일 주소**(JSON 내 `client_email` 값)를 추가합니다.
4. 권한을 **'편집자'**로 설정하고 저장합니다.

---

## ❓ 자주 묻는 질문

### Q: "Permission Denied" 오류가 발생합니다.
A: 5단계의 '구글 시트 공유' 설정을 확인하세요. 서비스 계정 이메일이 해당 시트의 편집자로 등록되어 있어야 합니다.

### Q: 여러 시트에 내보내고 싶습니다.
A: 내보내려는 모든 구글 시트 파일에 5단계 작업을 동일하게 수행해주시면 됩니다.

### Q: 인증 정보를 변경하고 싶습니다.
A: 설정 페이지에서 기존 연동을 해제하거나, 새 JSON 내용을 붙여넣고 업데이트하면 즉시 변경됩니다.
