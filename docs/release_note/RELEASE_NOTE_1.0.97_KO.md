# Release Note - v1.0.97

## [1.0.97] - 2026-07-22

작성: 2026-07-22 17:33 KST

테스트케이스와 자동화(JUnit) 결과를 서로 연결하고, 그 관계를 양쪽 화면에서 오갈 수 있게 한 기능 릴리즈입니다. 자동화 여부를 켠 테스트케이스에서 관련 테스트케이스와 실제 JUnit 케이스를 연결할 수 있고, 연결된 항목은 클릭 한 번으로 해당 페이지로 이동합니다. 링크가 어느 쪽에서든 끊기거나 삭제될 때 반대쪽에 죽은 링크가 남지 않도록 정합성도 함께 정리했습니다.

### 주요 변경 사항

#### ✨ 새 기능

* **연결된 테스트케이스 / 자동화(JUnit) 케이스**: 테스트케이스에서 "자동화 여부"를 켜면 (1) 같은 프로젝트의 다른 테스트케이스와 (2) 이 테스트케이스를 자동화한 실제 JUnit 케이스를 연결할 수 있습니다. 각 링크는 클릭하면 해당 상세 페이지로 이동합니다.
* **역방향 표시**: 테스트케이스 화면에는 "이 테스트케이스를 연결한 케이스"가, JUnit 결과 화면에는 "연결된 테스트케이스"가 각각 표시되어 어느 쪽에서 봐도 관계를 파악하고 이동할 수 있습니다.

#### 🔧 정합성·안정성

* **삭제 시 링크 정리**: 테스트케이스를 삭제하면 내가 건 링크와 나를 가리키던 링크를 모두 정리하고, JUnit 결과를 삭제하면 그 결과의 케이스를 연결하던 테스트케이스 링크도 함께 정리합니다. 어느 쪽에도 죽은 링크(dangling)가 남지 않습니다.
* **링크 보존 규칙**: 저장 요청에서 링크 필드를 아예 보내지 않으면 기존 링크를 유지하고, 빈 값으로 보내면 전체 해제로 처리합니다. 일부 자동 적재 경로에서 링크가 의도치 않게 지워지던 문제를 막습니다.
* **삭제 원자성**: JUnit 결과 삭제 중 오류가 나면 부분 삭제 없이 전체가 롤백됩니다.

#### 🖥️ UI

* **서버 시간·버전 표시 정리**: 좌측 하단 시간·버전 위젯을 기본 접힘(작은 아이콘)으로 바꾸고 관리자에게만 노출합니다. 상단 버전 표기는 헤더 배경에 맞춰 항상 보이도록 색상을 정리했습니다.

### 업그레이드 안내

* DB 마이그레이션 스크립트는 없습니다. 연결 정보를 담는 테이블 2개(`testcase_linked_test_cases`, `testcase_linked_junit_cases`)는 앱 기동 시 `ddl-auto=update`로 자동 생성됩니다.
* 별도 환경변수 변경 없이 이미지 교체만으로 적용됩니다.
