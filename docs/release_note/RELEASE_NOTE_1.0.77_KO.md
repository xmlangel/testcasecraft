# Release Note - v1.0.77

## [1.0.77] - 2026-05-22

안녕하세요! 이번 1.0.77 업데이트에서는 **RAG 채팅 API 외부 연동 인증(401/403) 오류 예외 핸들링을 정밀 고도화**하고, 사용자가 화면에서 즉각 대응할 수 있도록 **명확하고 친절한 조치 가이드를 한국어와 영어로 구현**하였습니다.

### 주요 변경 사항

#### 🛠️ 외부 LLM 클라이언트 계층 예외 정교화
* RAG(검색 증강 생성) 기반 채팅 시 외부 LLM API(OpenWebUI, OpenAI, OpenRouter, Perplexity, Ollama)와의 통신 도중 만료되거나 잘못 등록된 API Key로 인해 발생할 수 있는 `401 Unauthorized` 또는 `403 Forbidden` 인증 에러를 정밀하게 가로챕니다.
* 기존의 모호했던 내부 500 에러("Failed to call API") 대신, HTTP 응답 코드를 조기에 분기 처리하는 `WebClientResponseException` 핸들러를 Catch 블록 최상단에 배치하여 신속히 대응합니다.
* 예외 상황 발생 시 사용자 친화적인 메시지가 포함된 `LlmClientException`을 발생시켜 화면단에 직관적인 조치 방법을 즉시 표출합니다.

| 대상 컴파일 파일 | 개선 및 변경 사항 |
| :--- | :--- |
| **OpenWebUIClient.java** | 만료/잘못된 API Key로 인한 401 에러 감지 시 한글 안내 및 복호화 예외 로깅 강화 |
| **OpenAIClient.java** | 비스트리밍(`chat`) 및 스트리밍(`chatStream`) 연동 전송에 정교한 API 오류 격리 적용 |
| **OpenRouterClient.java** | OpenRouter 엔드포인트 응답 지연 및 비정상 API Key 예외 상황 캐치 추가 |
| **PerplexityClient.java** | Perplexity AI 연동 규격에 맞춰 HTTP 응답 예외 바인딩 고도화 |
| **OllamaClient.java** | Ollama API의 인증 요구 상황 시 적합한 한국어 가이드 메시지 반환 핸들러 추가 |

### 💡 관리자 및 사용자 조치 가이드

RAG 채팅 기능을 활용하는 도중 아래와 같은 팝업이나 토스트 에러가 표출되는 경우, 다음과 같이 직관적으로 해결하실 수 있습니다.

> **💡 에러 발생 시 예시 화면 메시지:**
> *"API 인증에 실패했습니다 (401/403). 등록된 API Key가 올바르고 만료되지 않았는지 확인해 주세요."*

1. **원인:** 데이터베이스 혹은 설정에 등록된 해당 LLM 제공자용 API Key가 만료되었거나, 토큰 값이 잘못 입력되었습니다.
2. **조치 방법:**
   - 시스템 관리자 콘솔 또는 LLM 연동 설정 메뉴로 이동합니다.
   - 연동하려는 외부 서비스(예: OpenAI, OpenWebUI) 웹사이트에서 유효한 신규 API Key를 발급받습니다.
   - 설정 입력창에 해당 API Key를 교체 등록한 후 저장하면, 시스템 재시작 없이 실시간으로 RAG 채팅 기능이 정상 복구됩니다.
