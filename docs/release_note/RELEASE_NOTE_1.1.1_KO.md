# Release Note - v1.1.1

## [1.1.1] - 2026-07-04

안녕하세요! 1.1.1은 minor 버전이 올라간 릴리즈입니다. **AgensGraph 기반 그래프 데이터 모델**이 새로 들어갔습니다. 테스트 케이스·실행·결과·오류 사이의 관계를 그래프로 눈으로 확인할 수 있고, 테스트 케이스 자체를 그래프 방식으로 만들거나 기존 케이스를 전환할 수도 있습니다. 기존 데이터와 화면은 그대로 동작합니다 — 그래프는 기능 플래그로 켜는 추가 기능입니다.

### 주요 변경 사항

#### 🕸️ 그래프 뷰 (신규)
* 프로젝트 헤더의 **그래프 아이콘(Hub)** 으로 진입하는 새 화면 `/graph` 이 생겼습니다. 세 가지 탭을 제공합니다.
  * **구조 그래프** — 프로젝트 → 폴더 → 케이스 → 계획 → 실행 → 결과의 관계망
  * **오류 클러스터** — 같은 실패 원인(FailureType)이 어느 케이스·실행까지 퍼져 있는지 허브 형태로 표시. JUnit 업로드의 실패 시그니처(failureType + 메시지 첫 줄)로 묶입니다.
  * **케이스 이웃** — 특정 케이스에서 1~3단계 연결된 정점·간선 탐색
* 포스(fcose)/계층(dagre) 레이아웃 전환, 노드 클릭 시 우측 상세 패널.
* 그래프 데이터는 **동기화 API**(`POST /api/graph/sync?projectId=`, 프로젝트 관리 권한)로 적재합니다. 멱등이라 언제든 다시 돌려도 됩니다.

#### 🧩 그래프 테스트 케이스 (신규)
* 테스트 케이스를 **세 가지 방식** 중에서 선택해 다룰 수 있습니다.
  * **기본** — 지금까지의 선형 스텝 방식 그대로 (기본값, 아무 것도 달라지지 않음)
  * **그래프** — 스텝을 노드 체인으로 저작 (`/graph-tc/{id}/edit` 편집기)
  * **기본 → 그래프 전환** — 기존 케이스를 비파괴적으로 전환 (`POST /api/graph/testcase/{id}/convert`), 폴더/프로젝트 단위 일괄 전환도 지원 (`bulk-convert`)
* **관계형 미러 보장**: 그래프로 만든 케이스도 기존 스프레드시트·폼에서 항상 선형 스텝으로 조회됩니다. 그래프 모드에서는 그래프가 원본이고 스텝 표는 읽기 전용 미러로 자동 갱신됩니다.
* 역전환(`revert`)하면 미러가 다시 원본이 됩니다.

#### 🗄️ 배포 구성
* docker-compose 에 **agensgraph 서비스**(`skaiworldwide/agensgraph:v2.16.0-bookworm`, PostgreSQL 16.9 기반)가 추가되었습니다. 메인 DB(pg18)와는 별도 인스턴스이며 기본 포트는 **5437** 입니다.
* 그래프 스키마(라벨 36종 + 인덱스)는 최초 기동 시 `init-scripts-graph/` 가 자동 생성합니다.

---

### ⚙️ 활성화 방법

그래프 기능은 **기본 꺼짐**입니다. `.env` 또는 환경변수에 다음을 설정하면 켜집니다.

```bash
FEATURES_GRAPH_ENABLED=true
GRAPH_DB_PASSWORD=<비밀번호>
# 필요 시: GRAPH_DB_URL, GRAPH_DB_USERNAME, GRAPH_DB_PORT(기본 5437)
```

켠 뒤 프로젝트 관리자 권한으로 `POST /api/graph/sync?projectId=<프로젝트ID>` 를 한 번 실행하면 그래프 뷰에 데이터가 나타납니다.

### ⚠️ 버전 호환성 (중요)

| 방향 | 지원 여부 |
|------|-----------|
| 1.0.x 데이터 → **1.1.1** | ✅ 그대로 동작합니다. 기존 케이스는 전부 "기본" 모드로 인식되고, 스키마 변경은 컬럼 추가뿐입니다. |
| **1.1.1** → 1.0.x 다운그레이드 | ❌ 공식 지원하지 않습니다. |

1.1.1에서 **그래프 모드로 만든/전환한 케이스**는 1.0.x로 되돌리면 그래프 구조를 볼 수 없습니다 (관계형 미러의 선형 스텝만 조회 가능, 크래시는 없음). 다운그레이드 전에는 해당 케이스들을 역전환(`revert`)해 두는 것을 권장합니다.

### 사용자 조치 가이드

* **그래프 기능을 쓰지 않는 경우** — 아무 조치가 필요 없습니다. 플래그가 꺼져 있으면 1.0.x와 동일하게 동작하고 agensgraph 컨테이너 없이도 기동합니다.
* **그래프 기능을 켜는 경우** — 위 활성화 방법대로 env 설정 후 `docker compose up -d agensgraph` 로 그래프 DB를 먼저 띄우고 앱을 재시작하세요.
