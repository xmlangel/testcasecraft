# Release Note - v1.0.95

## [1.0.95] - 2026-07-20

안녕하세요! 1.0.95는 코드 리뷰(dev-review R2) 후속으로 진행한 **보안 하드닝**과 **기능·정확성 버그 수정**을 담은 릴리즈입니다.

### 주요 변경 사항

#### 🔒 보안 하드닝

> **보안 하드닝이란?** 새 기능을 넣는 게 아니라, 원래 열리면 안 되는 접근·주입 경로를 찾아 막고 시스템을 더 단단하게 만드는 방어 위주의 수정입니다. "되던 게 더 되게"가 아니라 "되면 안 되던 게 안 되게" 만드는 작업입니다.

무인가 접근(IDOR/BOLA)·인젝션·SSRF 등 P0/P1 취약점을 다수 봉쇄했습니다. 주요 영역: 객체 수준 인가 부착(RAG·첨부·권한), Jira 연동 SSRF·JQL·JSON 주입 차단, RAG의 LLM 생성 SQL 경계 강화, 서비스 API 키 해시 저장(SHA-256), 운영 프로파일 fail-closed(기본 암호화키·관리자 자격증명·SSL 우회), RateLimiter 방어(XFF 우회·OOM·XSS), CORS/Swagger/Actuator 노출 차단, 인가 거부 403 정본화.

#### 🐞 기능·정확성 버그 수정

* **최상위(폴더 밖) 노드 순서 변경 실패 수정**: 폴더 안은 되는데 최상위 폴더·루트 케이스만 드래그 이동이 실패하던 문제를 고쳤습니다(형제 조회의 루트 미처리). 루트 조회를 현재 프로젝트로 한정해 프로젝트 간 순서 오염도 예방했습니다.
* **진행률 오집계 수정**: 미실행(NOT_RUN)이 완료로 잡히던 드리프트를 프론트·백엔드 양쪽에서 수정했습니다.
* **RAG·화면 수정**: RAG 채팅 생성/수정(snake_case 본문)·스레드 PATCH 400, 서비스 API 키 목록 컬럼 공백·헤더 문구, 리포트 null NPE를 정정했습니다.

#### 🧹 내부 개선

판정 상태 `TestResultStatus` enum 정본화, 운영 프로파일·권한 투영 공통화, 미사용 코드 제거, 회귀 가드·통합 테스트 27건 복구(트리 수정용 서비스·실 DB 테스트 포함).

### 업그레이드 안내

* DB 마이그레이션 없음 — 이미지 교체만으로 적용됩니다.
* **운영 주의**: 기본 암호화키·관리자 자격증명·SSL 우회가 운영 프로파일에서 fail-closed로 동작합니다. 관련 보안 환경변수는 저장소의 운영 배포 보안 문서를 확인하세요.
