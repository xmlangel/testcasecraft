# Release Note - v1.0.93

## [1.0.93] - 2026-07-02

안녕하세요! 1.0.93은 **테스트 케이스를 다른 프로젝트로 이동·복사**하는 기능을 새로 선보이고, **인가(권한) 보안을 대폭 강화**한 릴리즈입니다. 여기에 다국어(i18n) 문구를 전면 정리하고, 테스트 결과 리포트의 가독성을 높였습니다.

### 주요 변경 사항

#### 📦 다른 프로젝트로 테스트 케이스 이동·복사 (신규)

* 트리에서 테스트 케이스(및 폴더)를 **다른 프로젝트로 이동하거나 복사**할 수 있습니다. **이동**하면 연결된 **테스트 결과도 함께** 옮겨지고(대상 프로젝트에 대응 실행을 새로 만들어 결과를 미러링), **복사**는 **케이스만 복제**하고 결과는 가져오지 않습니다.
* 백엔드·프론트엔드 트리 UI에 더해 **MCP 도구(`testcase_move_to_project`, `testcase_copy_to_project`)**로도 동일하게 수행할 수 있습니다.
* 이동·복사 과정에서 발견된 구조 문제를 함께 바로잡았습니다 — 전체 선택이 폴더를 포함하도록 수정(구조 평탄화 방지), 대상 폴더·루트의 **맨 마지막 순서로 삽입**(중간 삽입 버그 수정), 입력이 자식-우선으로 들어와도 폴더 구조가 평탄화되지 않도록 수정.

#### 🔒 인가(권한) 보안 하드닝 (수정)

* **스케줄러 설정 변경 비인증 접근 차단 (CRITICAL)**: 인증 없이 스케줄러 설정을 변경할 수 있던 경로를 차단했습니다.
* **프로젝트 단위 인가 추가**: TestCase·TestPlan·TestExecution CRUD와 RAG 문서 분석·임베딩·삭제에 프로젝트 소속 검증을 추가해, 다른 프로젝트의 리소스에 접근할 수 없도록 했습니다.
* **RBAC 인가 하드닝 + 통합 테스트**: 역할 기반 접근 제어를 정비하고 인가 통합 테스트를 추가했습니다.
* **403 권한 오류 사용자 친화 메시지**: 권한이 없을 때 원인을 알기 쉬운 메시지로 안내합니다.

#### 🌐 다국어(i18n) 정리 (개선)

* 컴포넌트·유틸 레벨에 남아 있던 **하드코딩 한국어 문구를 `t()`로 래핑**하고 한/영 시드를 보강했습니다(유틸 메시지 36키 포함).
* **미사용 번역 키 719개를 제거**해 번역 시드를 정리했습니다.

#### 🗑️ 테스트 케이스 개별 폼 삭제 버튼 (신규·개선)

* 테스트 케이스 개별 편집 폼에서 바로 **삭제**할 수 있는 버튼을 추가했습니다.
* 삭제 확인을 **트리 삭제와 동일한 확인 다이얼로그**로 통일했습니다 — 기존의 브라우저 기본 팝업(`window.confirm`) 대신, 삭제 대상(ID·이름)을 표로 보여주고 백엔드 인가 오류(403 등) 메시지를 그대로 노출하는 방식으로 일관성을 맞췄습니다.

#### 📊 테스트 결과 리포트 개선 (개선)

* 테스트 결과에 **실패·성공·차단 분류 칩**을 추가하고, HTML/PDF 리포트에서 각 판정을 **색상으로 구분**해 한눈에 파악할 수 있습니다.
