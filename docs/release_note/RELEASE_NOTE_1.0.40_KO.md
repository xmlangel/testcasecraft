# 릴리스 노트 - v1.0.40

## 🚀 주요 변경 사항

### ⚡ 초기 로딩 시인성 및 성능 개선
- **스플래시 화면(Splash Screen) 도입**: 애플리케이션 초기 구동 시 사용자에게 진행 상황을 보여주는 공식 스플래시 화면을 추가했습니다.
- **프로그레스 바 시각화**: i18n 초기화 및 핵심 리소스 로딩 과정을 프로그레스 바를 통해 실시간으로 확인할 수 있습니다.
- **로딩 프로세스 최적화**: 메인 스크립트 로딩 방식을 개선하여 초기 구동 안정성을 높였습니다.

### 📋 JUnit 테스트 결과 상세 표시 강화
- **전 상태 상세 정보 추출**: 기존 실패 케이스에만 국한되었던 상세 정보 파싱 범위를 확대하여, **성공(Passed)**, 에러(Error), 스킵(Skipped)된 모든 테스트의 상세 단계와 결과를 확인할 수 있습니다.
- **상태 기반 UI 테마 적용**: 통과된 테스트의 상세 결과는 초록색(Success) 계열로 표시하여 실패 결과와 시각적으로 명확히 구분되도록 개선했습니다.
- **메타데이터 파싱 일반화**: XML 구조와 상관없이 기대 결과 및 실제 결과 태그를 일관되게 추출하도록 로직을 강화했습니다.

## 🛠 버그 수정 및 최적화
- **무한 로딩 이슈 해결**: i18n 초기화와 스플래시 화면 종료 시점의 경합 조건(Race Condition)으로 인해 발생하던 무한 로딩 문제를 수정했습니다.
- **의존성 최적화**: 미사용 `@toast-ui/editor` 라이브러리를 제거하여 번들 크기를 최적화했습니다.
- **앱 초기화 안정성**: AppWrapper를 통한 라이프사이클 관리로 초기 로딩 안정성을 강화했습니다.
