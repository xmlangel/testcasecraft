# Release Note - v1.0.85

## [1.0.85] - 2026-06-09

안녕하세요! 1.0.85는 사용성 중심 업데이트입니다. 자주 보는 테스트 케이스를 개인 **북마크**로 모아 둘 수 있는 즐겨찾기 기능을 새로 추가하고, 테스트 실행 화면을 **자동 새로고침**과 **필터 패널**로 한층 편리하게 다듬었습니다.

### 주요 변경 사항

#### ⭐ 테스트 케이스 즐겨찾기 & 북마크 (신규)

자주 참조하는 케이스를 개인 북마크로 정리하는 기능을 추가하였습니다. 북마크는 **나에게만 보이는 개인 정보**로, 다른 사용자의 화면에는 영향을 주지 않습니다.

* **즐겨찾기 토글**: 케이스 목록의 각 행에 별 아이콘(☆/★)이 추가되어, 한 번의 클릭으로 즐겨찾기에 담거나 뺄 수 있습니다.
* **북마크 화면**: 헤더의 ☆ 버튼으로 북마크 화면(`/projects/{projectId}/bookmarks`)에 진입합니다. 좌측의 **모음(컬렉션)** 목록과 우측의 케이스 목록(케이스명·우선순위·메모·동작)으로 구성됩니다.
* **모음 관리**: 케이스를 주제별로 묶는 모음을 **만들고·이름 변경·삭제**할 수 있습니다. 기본 **즐겨찾기** 모음은 항상 존재하며 삭제할 수 없습니다.
* **개인 메모**: 각 케이스에 "회귀 때 꼭 확인" 같은 개인 메모를 남길 수 있습니다.
* **읽기 전용**: 북마크 화면은 모음 정리와 메모만 다루며, 케이스 본문 편집은 기존처럼 테스트케이스 화면에서 수행합니다.

#### 🔄 테스트 실행 목록 자동 새로고침 + 필터 패널 (개선)

* **자동 새로고침**: 테스트 실행 목록이 **약 20초마다 자동으로 갱신**되어, 다른 팀원이 기록한 진행 상황이 새로고침 없이 반영됩니다. 다른 탭을 보고 있는 동안에는 갱신이 멈춰 불필요한 요청이 발생하지 않으며, 화면으로 돌아오면 다시 갱신됩니다. 목록 상단의 **[새로고침]** 버튼으로 언제든 즉시 갱신할 수도 있습니다.
* **필터 패널**: 실행 상세의 케이스 목록 위에 필터 패널이 추가되어 **결과**(PASS·FAIL·BLOCKED·NOT RUN)·**우선순위**·**실행자**·**실행일자**·**JIRA 아이디**·**노트** 조건으로 케이스를 추려 볼 수 있습니다. 필터가 적용되면 **적용 중** 표시가 나타나고, **[초기화]** 로 모든 조건을 한 번에 해제합니다.
* **검색어 보존**: 실행 목록의 제목 검색어가 프로젝트별로 기억되어, 화면을 다시 열어도 직전 검색 상태가 유지됩니다.

#### 🔐 보안 코드 정리

* Spring Security 6.4+ 에서 deprecated 된 `DaoAuthenticationProvider` API 사용을 정리하였습니다(생성자 주입으로 전환). 동작 변화는 없으며 컴파일 경고를 해소합니다.

### 📖 사용자 매뉴얼

* 한국어/영문 매뉴얼에 위 변경 사항을 반영하였습니다 — §3-1 헤더 ☆ 버튼, §4-7 "북마크 & 즐겨찾기" 신규 섹션, §8 자동 새로고침·필터 패널 안내 및 실행 화면 캡처 갱신.

### 업그레이드 안내

* 별도 마이그레이션 없이 이미지 교체만으로 적용됩니다.
* 북마크 데이터는 사용자별로 저장되며, 기존 데이터에 영향을 주지 않습니다.
* 1.0.84의 보안 업데이트(취약점 해소, JWT_SECRET 검증·자동 생성)는 [RELEASE_NOTE_1.0.84_KO.md](RELEASE_NOTE_1.0.84_KO.md)를 참고하세요.
