# 릴리스 노트 - v1.0.41

## 🚀 주요 변경 사항

### 📦 JUnit 업로드 및 파일 스토리지 개선

- **드래그 앤 드롭 업로드 수정**: JUnit 결과 파일 업로드 시 드래그 앤 드롭 기능이 정상적으로 작동하도록 수정했습니다.
- **MinIO 스토리지 통합**: 첨부파일 및 결과 데이터를 MinIO(S3 호환 스토리지)에 저장하도록 통합하여 파일 관리 안정성을 높였습니다.

### 🔍 로그 분석 및 메타데이터 추출 강화

- **레거시 단계 파서(Legacy Steps Parser) 도입**: 로그 내 특수한 패턴의 테스트 단계를 자동으로 파싱하는 기능을 추가했습니다.
- **메타데이터 추출 로직 개선**: 로그 데이터에서 기대 결과 및 실제 결과를 더 정확하게 추출하도록 알고리즘을 강화했습니다.
- **진단 로깅 강화**: 메타데이터 파싱 과정 및 시스템 출력 길이에 대한 상세 로그를 추가하여 분석 편의성을 높였습니다.

### 🎨 UI/UX 및 테마 일관성 개선

- **다크 테마 미리보기 수정**: 코드 및 텍스트 미리보기 창에서 하드코딩된 배경색을 제거하고 다크 테마에 맞게 `action.hover` 배경을 적용했습니다.
- **컴포넌트 구조 최적화**: 미리보기 영역의 `pre` 태그를 MUI `Box` 기반으로 리팩토링하여 테마 변수가 일관되게 적용되도록 개선했습니다.
- **UI 컴포넌트 추가**: `TestExecutionForm`에 MUI `Chip` 컴포넌트를 도입하여 가독성을 높였습니다.

## 🛠 버그 수정 및 최적화

- **LinearProgress 참조 오류 수정**: `TestExecutionForm` 컴포넌트에서 `LinearProgress` 임포트가 누락되어 발생하던 런타임 오류를 수정했습니다.
- **의존성 업데이트**: 프론트엔드 라이브러리 및 의존성 패키지를 최신 버전으로 업데이트했습니다.
