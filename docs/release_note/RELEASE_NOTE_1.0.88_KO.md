# Release Note - v1.0.88

## [1.0.88] - 2026-06-14

안녕하세요! 1.0.88은 **초기 화면 진입 성능**에 집중한 안정화 업데이트입니다. 로그인 직후·화면 전환 시 같은 API가 두 번씩 호출되던 여러 지점을 정리하여 불필요한 네트워크 요청과 서버 부하를 줄였습니다. 기능 변화는 없으며, 체감 로딩이 가벼워집니다.

### 주요 변경 사항

#### ⚡ 초기 API 중복 호출 제거 (성능)

화면 진입 시 동일한 요청이 2회 이상 발생하던 문제들을 일괄 정리했습니다.

* **Provider 트리 2중 중첩 제거**: 최상위에서 Context Provider가 이중으로 마운트되어 모든 Context의 초기 fetch(`rag/status`·`auth/me`·`preferences` 등)가 통째로 중복 실행되던 구조를 제거했습니다.
* **fetch effect 재실행 폭주 제거**: 매 렌더마다 새로 생성되던 불안정한 함수 참조(`api` 등)가 의존성으로 물려 인증 관련 fetch effect가 반복 실행되던 문제를 ref 고정으로 해결했습니다.
* **조직 멤버 목록(`/api/organizations/{id}/members`)**: 같은 조직에 속한 프로젝트 카드들이 각각 동일한 조직 멤버 API를 호출하던 문제를 조직 단위 캐시 + 진행 중 요청 공유로 단일 호출화했습니다.
* **사용자 환경설정(`/api/auth/me/preferences`)**: 설정 키마다 전체 환경설정을 따로 내려받던 것을, 모든 사용처가 단일 요청을 공유하도록 정리했습니다.
* **RAG 상태(`/api/system-settings/rag/status`)**: 개발 모드(StrictMode) 이중 마운트나 재마운트 시 중복 호출되던 것을 진행 중 요청 공유로 1회만 발생하도록 했습니다.

#### 🛠️ PWA 아이콘 401 수정 (개선)

* 매니페스트와 `apple-touch-icon`이 참조하는 `logo192.png`가 보안 필터에 막혀 **401 (Unauthorized)** 로 응답하며 콘솔 오류와 아이콘 다운로드 실패를 일으키던 문제를 수정했습니다. 정적 리소스 허용 목록에 PWA 아이콘 경로를 추가했습니다.
