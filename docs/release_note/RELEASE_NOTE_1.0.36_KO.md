# 릴리스 노트 - v1.0.36

## 🚀 주요 변경 사항

### ⚙️ 빌드 및 배포 자동화 (CI/CD)
- **GitHub Actions 통합**: Docker 이미지를 자동으로 빌드하고 배포하는 워크플로우를 구축했습니다.
- **태그 기반 빌드**: 태그 이름(`v*-app`, `v*-rag`, `v*-all`)에 따라 빌드 대상을 유연하게 선택할 수 있습니다.
- **자동 릴리스 생성**: 태그 푸시 시 `docs/release_note/`의 파일을 참조하여 GitHub Release를 자동으로 생성합니다.

### ✅ JUnit 및 테스트 검증 강화
- **JUnit XML 구조화 개선**: 테스트 단계별 실행 정보, 기대 결과, 실제 결과를 개별 XML 엘리먼트로 구조화하여 테스트 결과의 가독성과 분석 효율을 높였습니다.
- **오류 분석 고도화**: 테스트 실패 시 원인 파악이 용이하도록 상세 스텝 정보를 포함하는 로직을 강화했습니다.

### 📚 문서화 및 가이드
- **릴리스 가이드 추가**: 로컬에서의 버전 업데이트 및 빌드 정합성 확보를 위한 상세 가이드를 `docs/dev_md/14_릴리즈_절차_및_방법_가이드.md`에 추가했습니다.
- **GitHub Actions 기술 문서**: 자동화 워크플로우의 구조와 사용법에 대한 기술 문서를 완비했습니다.

### 🛠 시스템 개선
- **빌드 스크립트 고도화**: `build-and-push-multiplatform.sh` 스크립트에 비대화형(CI) 모드 및 인자 처리 기능을 추가했습니다.
- **메타데이터 동기화**: 빌드 프로세스 개선을 통해 `package-lock.json` 등 주요 메타데이터의 버전 정합성을 강화했습니다.
