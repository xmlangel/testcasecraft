# 릴리스 노트 - v1.0.30

## 🚀 주요 변경 사항

### 🔍 테스트 관리 및 검색

- **테스트 케이스 이름 필터**: 테스트 실행 뷰에서 테스트 케이스 이름으로 즉시 검색할 수 있는 필터 기능과 관련 UI를 추가했습니다.
- **필드 확장**: 테스트 케이스에 사후 조건(Post-condition), 테스트 기법(Test Technique), 자동화 상태(Automation Status) 필드를 추가하여 더 상세한 관리가 가능해졌습니다.
- **최대 단계 확장**: 테스트 케이스 단계를 기존보다 확장된 최대 20단계까지 지원합니다.

### 🤖 LLM 및 RAG 연동

- **RAG API 최적화**: API 호출 시 사용하는 쿼리 파라미터(`project_id`)를 카멜 케이스(`projectId`)로 표준화하고 성능을 개선했습니다.
- **기본 LLM 템플릿 제공**: 서버에서 기본 LLM 템플릿을 제공하는 전용 API 엔드포인트를 추가하고 프론트엔드에서 이를 가져와 사용하도록 연동했습니다.

### 💾 데이터 처리 및 보안

- **계층형 배치 저장 (ICT-415)**: 폴더와 테스트 케이스 간의 부모-자식 의존성을 지능적으로 해결하여 저장하는 레이어별 배치 저장 로직을 구현했습니다.
- **보안 핸들러 강화**: Spring Security의 401/403 에러 핸들링을 커스터마이징하고, 프론트엔드 Axios 인터셉터를 보완하여 권한 부족 응답에 더 정교하게 대응합니다.

### 📂 폴더 및 태그 관리

- **태그 전파 로직**: 폴더에 태그를 설정할 때 하위 아이템으로 태그가 전파되도록 기능을 개선하고, 태그 관리 전용 UI를 추가했습니다.
