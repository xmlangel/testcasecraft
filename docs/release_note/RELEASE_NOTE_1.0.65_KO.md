# Release Note - v1.0.65

## [1.0.65] - 2026-04-14

v1.0.61 이후의 주요 기능 개선 및 시스템 최적화 사항을 통합한 릴리즈 노트입니다.

### 주요 변경 사항

#### 🚀 PDF 리포트 수출 기능 고도화 (Advanced PDF Export)
- **리포트 스타일링 강화**: PDF 방향(가로/세로) 선택 기능, 커스텀 헤더/푸터, KPI 요약 섹션을 추가하여 전문적인 리포트 출력이 가능해졌습니다.
- **브랜딩 및 가시성 개선**: 푸터에 커스텀 브랜딩을 적용하고, 테스트 플랜 및 실행 상세 정보를 포함하는 Sticky 페이지 헤더를 도입하였습니다.
- **사용성 개선**: 테스트 결과 테이블의 툴바를 리팩토링하고 컬럼 메뉴의 사용성을 개선하여 리포트 구성이 더욱 편리해졌습니다.

#### 🛠️ 백엔드 아키텍처 및 성능 최적화 (Backend & Performance)
- **DTO 리팩토링 (Lombok 도입)**: 모든 DTO 클래스에 Lombok을 적용하여 보일러플레이트 코드를 제거하고, Bean Validation 제약 조건을 추가하여 데이터 정밀도를 높였습니다.
- **데이터 조회 최적화**: 테스트 결과 조회 시 데이터베이스 레벨의 중복 제거와 서버 사이드 페이지네이션을 구현하여 대용량 데이터 처리 성능을 개선하였습니다.
- **빌드 및 서비스 안정성**: Gradle 빌드 구성을 개선하고, 서비스 초기화 로직을 최적화하여 애플리케이션 시작 및 운영 안정성을 확보하였습니다.

#### 🔍 테스트 결과 필터링 및 관리 기능 강화 (Test Result Management)
- **최신 결과만 보기 필터**: 복잡한 테스트 히스토리 중 각 케이스별 가장 최신 결과만 필터링하여 볼 수 있는 기능을 서버 사이드 및 클라이언트 UI에 구현하였습니다.
- **날짜 파싱 호환성 개선**: 브라우저 간 호환성을 위해 날짜 파싱 로직을 개선하여 정확한 시간 정보를 제공합니다.

#### ✨ 시스템 UI 및 보안 개선 (System UI & Security)
- **Header 정보 표시**: 애플리케이션 헤더에 현재 버전 정보와 서버 시간을 표시하여 사용자가 환경 정보를 즉시 확인할 수 있도록 개선하였습니다.
- **보안 및 접근성 최적화**: RAG 상태 엔드포인트의 보안 설정을 업데이트하고, 인증이 필요 없는 엔드포인트에 대한 `skipAuth` 옵션을 도입하여 유연한 접근 제어를 지원합니다.

#### 📊 탐색적 테스팅(Exploratory Testing) UI 전면 개편
- **카드 레이아웃 도입**: 세션 목록을 직관적인 카드 방식으로 디자인하여 가독성과 접근성을 높였습니다.
- **세션 추적 및 자동화**: 세션 승인 및 중단 과정을 추적하고, 진행 상황을 시각적으로 보여주는 프로그레스 바를 추가하였습니다.
- **상태 유지 기능**: 사용자의 탐색적 세션 탭 정보를 URL 기반으로 저장하여 브라우저 새로고침이나 이동 시에도 작업 컨텍스트가 유지되도록 하였습니다.

#### 🎨 데이터 모델 및 레이아웃 표준화
- **테스트 케이스 트리 정렬**: 트리의 라인과 컬럼 너비를 고정하여 시각적 일관성을 확보하고 대규모 트리 구조에서도 가독성이 유지되도록 개선하였습니다.
- **DB 안정성 강화**: 스키마 초기화 로직을 정교화하고, 데이터베이스 제약 조건 위반에 대한 전역 예외 처리를 도입하여 안정적인 데이터 관리를 가능케 하였습니다.
