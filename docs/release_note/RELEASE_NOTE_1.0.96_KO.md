# Release Note - v1.0.96

## [1.0.96] - 2026-07-20

1.0.96은 1.0.95 보안 하드닝이 남긴 두 회귀를 잡는 핫픽스입니다. 둘 다 운영에 새 환경변수를 넣지 않으면 앱이 멈추던 문제라, 설정을 강제하지 않고도 정상 동작하도록 고쳤습니다. 하드닝 취지(취약한 기본값 차단)는 그대로 둡니다.

### 주요 변경 사항

#### 🐞 회귀 수정

* **화면이 안 뜨던 문제(정적 자산 403) 해결**: 1.0.95에서 CORS 허용 출처가 와일드카드(`*`)에서 환경변수 지정 방식으로 바뀌면서, `APP_CORS_ALLOWED_ORIGINS`를 설정하지 않은 배포에서는 서비스 자기 도메인조차 허용 목록에 없어 JS/CSS 로드가 403으로 거부됐습니다(빌드 산출물이 `crossorigin`으로 로드돼 브라우저가 CORS 요청을 보냄). 동일 출처(same-origin) 요청은 설정과 무관하게 항상 허용하도록 고쳐, 별도 설정 없이도 화면이 정상적으로 뜹니다. 교차 출처는 여전히 `APP_CORS_ALLOWED_ORIGINS`로만 허용하므로 하드닝은 유지됩니다.
* **암호화 키 미설정 시 컨테이너 unhealthy 해결**: `JIRA_ENCRYPTION_KEY`가 없으면 헬스 지표가 전체 상태를 DOWN으로 떨어뜨려 컨테이너가 unhealthy로 표시되고, 무관한 로그인·조회까지 영향을 받았습니다. 키가 없을 때는 앱 전체를 멈추지 않고 경고(WARN)로 표출하도록 바꿨습니다. 이 경우 Jira/LLM 설정 저장만 비활성화되고 나머지 기능은 정상 동작합니다.

### 업그레이드 안내

* DB 마이그레이션 없음 — 이미지 교체만으로 적용됩니다.
* 별도 환경변수 없이도 동작합니다. Jira/LLM 설정 저장을 쓰려면 `JIRA_ENCRYPTION_KEY`를, 교차 출처 접근을 허용하려면 `APP_CORS_ALLOWED_ORIGINS`를 설정하세요.
* 1.0.95 변경은 [RELEASE_NOTE_1.0.95_KO.md](RELEASE_NOTE_1.0.95_KO.md)를 참고하세요.
