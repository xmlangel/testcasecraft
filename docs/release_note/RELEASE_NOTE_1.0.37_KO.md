# 릴리스 노트 - v1.0.37

## 🚀 주요 변경 사항

### 🐳 Docker 이미지 최적화 및 경량화

- **주 애플리케이션 Alpine 전환**: Spring Boot 애플리케이션의 베이스 이미지를 Alpine Linux(`eclipse-temurin:21-jdk-alpine`)로 전환하여 이미지 크기를 300MB 이상 대폭 축소했습니다.
- **RAG 서비스 빌드 최적화**: ML 라이브러리(PyTorch 등)의 호환성을 고려하여 `slim` 이미지를 유지하되, 레이어 구조 재배치를 통해 캐시 효율성을 극대화했습니다.

### ⚡ 빌드 속도 개선 (CI/CD)

- **Buildx 레지스트리 캐시 도입**: `docker buildx build` 명령어에 레지스트리 캐시 옵션(`--cache-from`, `--cache-to`)을 추가했습니다.
- **반복 빌드 시간 단축**: 변경되지 않은 OS 패키지 및 라이브러리 레이어를 Docker Hub 캐시에서 재사용하여, 특히 arm64/x86 멀티 플랫폼 빌드 시간을 획기적으로 단축했습니다.

### ⚙️ 버전 관리 정책 업데이트 (RAG 서비스 제외)

- **RAG 서비스 독립 버전 관리**: 이제 일반적인 버전 업그레이드 시 `rag-service`의 버전은 변경되지 않습니다. RAG 서비스의 버전은 명시적으로 `-PtargetComponent=rag-service` 인자를 사용할 때만 업데이트됩니다.
- **시스템 정합성 유지**: 주 애플리케이션과 프론트엔드의 버전 업그레이드 주기를 RAG 서비스와 분리하여 빌드 효율성을 높였습니다.

### 🛠 시스템 안정성

- **프로세스 관리 개선**: `dumb-init`을 Alpine 환경에 맞게 적용하여 컨테이너 시그널 처리를 최적화했습니다.
- **패키지 매니저 보안 최적화**: Alpine 기반 환경에서 `apk`를 활용한 최소 패키지 구성을 통해 보안 취약점을 줄였습니다.
