# Release Note - v1.0.80

## [1.0.80] - 2026-06-05

안녕하세요! 이번 1.0.80 업데이트는 **테스트 실행 결과 입력의 데이터 안정성**에 집중하였습니다. "결과 페이지를 열어 보기만 했는데 기존 입력이 지워지거나 빈 NOT_RUN 레코드가 쌓이는" 문제를 근본적으로 해결하여, **사용자가 실제로 수정한 경우에만 저장이 일어나도록** 자동저장 동작을 재설계하였습니다. 함께 PDF 고급 내보내기 잘림 보정과 테스트케이스 입력 화면 레이아웃 개선도 포함되었습니다.

### 주요 변경 사항

#### 🛡️ 결과 입력: 보기만 하면 절대 저장되지 않습니다 (핵심)
* **사용자 수정 감지 게이트(userEdited)** 를 도입하여, 결과/노트/태그/JIRA 키를 **실제로 수정했을 때만** 자동저장(디바운스 저장·페이지 이탈 저장)이 활성화됩니다. 결과 입력 페이지를 열어 확인만 하고 닫으면 어떤 PUT/POST도 발생하지 않습니다.
* 페이지 진입 직후 초기 빈 폼 상태와 기존 결과 ID가 섞인 스냅샷이 "미저장 변경"으로 오판되어, **기존 결과가 빈 NOT_RUN으로 덮어써지던 버그**를 수정하였습니다 (React StrictMode 이중 마운트 / 케이스 전환 경합).
* 다음/이전 케이스 전환 시 폼을 **케이스별로 리마운트**(`key={testCaseId}`)하여, 이전 케이스의 입력 내용과 다음 케이스의 결과 ID가 섞여 저장되는 경합을 차단하였습니다.
* 결과를 변경하지 않고 저장/다음/Enter 를 눌러도 **이전 결과가 그대로 유지**되며, 새 레코드가 생성되지 않습니다.
* 신규 결과에서 아무 입력 없이(NOT_RUN·빈 노트·빈 태그) 저장해도 **빈 레코드가 생성되지 않습니다** — 자동저장·수동저장 모두 동일한 가드를 적용합니다.

#### 🧭 결과 표시 정확성 보정
* 풀페이지 결과 폼이 항상 **executedAt 기준 최신 결과**를 표시하도록 수정하였습니다.
* 플로팅 메뉴 **N(미실행) 버튼 값을 `NOTRUN` → `NOT_RUN`** 으로 교정하고, 과거 데이터의 레거시 표기(`NOTRUN`)도 호환 처리하였습니다.
* 이전 결과 목록에서 **현재 실행에 속한 행을 시각적으로 구분**하여, 어떤 결과가 이번 실행분인지 한눈에 확인할 수 있습니다.

#### 📄 PDF 고급 내보내기 잘림 해결
* 긴 테스트 단계(Step/Expected) 정보가 페이지 경계에서 잘리던 문제를 수정하여, **여러 페이지에 걸쳐 온전히 출력**됩니다.

#### 📐 테스트케이스 입력 화면 레이아웃
* 상세 탭을 **풀폭(1단) 레이아웃**으로 변경하여 긴 스텝·기대 결과를 더 넓게 편집할 수 있습니다.
* MUI v7 Grid API 마이그레이션으로 의도된 1단 레이아웃이 모든 화면 폭에서 일관되게 적용됩니다.

---

### 💡 사용자 조치 가이드

#### 결과 확인은 자유롭게, 저장은 수정했을 때만
* 결과 입력 페이지를 열어 내용을 확인하거나 다음/이전으로 이동만 하는 경우, 기존 결과는 절대 변경되지 않습니다.
* 결과 버튼(N/P/F/B)·노트·태그·JIRA 키를 수정하면 약 1.5초 후 자동저장되며, 헤더의 자동저장 상태 표시로 확인할 수 있습니다.

#### 과거에 생성된 빈 NOT_RUN 레코드
* 이전 버전에서 조회만으로 생성된 빈 NOT_RUN 레코드가 남아 있다면, 해당 케이스에서 실제 결과를 입력하면 그대로 갱신되어 정리됩니다.
