# Release Note - v1.0.92

## [1.0.92] - 2026-06-27

안녕하세요! 1.0.92는 **테스트 케이스 작성 편의성**을 한 단계 끌어올리고, 내부적으로는 **대규모 코드 정리와 테스트 보강**에 집중한 안정화 릴리즈입니다. 단계 입력 에디터가 너무 좁다는 피드백을 반영했고, 스프레드시트·JUnit·테스트 결과·PDF 내보내기 코드를 작은 단위로 쪼개 유지보수성과 회귀 안전성을 크게 높였습니다.

### 주요 변경 사항

#### 📝 테스트 단계 입력 에디터 최소 5줄 표시 (개선)

* 테스트 케이스 작성 화면에서 **단계(Step)·예상 결과(Expected) 입력 칸이 너무 좁다**는 피드백을 반영했습니다.
* 이제 각 입력 에디터가 **최소 5줄 높이**로 표시됩니다. 내용이 많아지면 기존처럼 최대 10줄까지 늘어나고 그 이상은 스크롤되며, 한 행의 높이는 Step·Expected 두 칸 중 큰 쪽에 맞춰집니다.

#### 🐞 편집 흐름 버그 수정 (수정)

* **단계 삭제 후 번호 꼬임**: 중간 단계를 지우면 남은 단계의 번호가 자동으로 다시 매겨지도록 했습니다.
* **폴더 삭제 시 유령 행**: 폴더를 삭제한 뒤 목록에 빈/잔여 행이 남던 문제를 수정했습니다.
* **부모 폴더 이동 시 태그 정리**: 케이스를 다른 폴더로 옮길 때 태그 정리가 누락되던 동작을 복구했습니다.
* **스프레드시트 저장 일원화**: 저장 경로를 단일 소스로 통합해 저장 중복·카운트 불일치를 방지했습니다.
* **실행 목록 필터 네비게이션**: 필터를 해제할 때 남아 있던 내비게이션 ID(navIds)를 제거해 다음/이전 이동이 어긋나지 않도록 했습니다.

#### 🧪 내부 품질 (리팩토링·테스트)

* 거대해진 화면 코드를 **작은 컴포넌트·커스텀 훅·순수 유틸**로 분리했습니다 — 스프레드시트(툴바·메뉴·행 선택·저장 유틸), JUnit 결과(통계 카드·실패/느린 테스트 탭·내보내기 훅), 테스트 결과 표(툴바·컬럼 설정·첨부/에러 상태), PDF 내보내기(텍스트·HTML 템플릿·섹션 빌더).
* 위 모듈 전반에 **vitest 단위·렌더 테스트를 대거 추가**했습니다 (utils·models·컴포넌트). 분리 과정에서 발견된 누락 상수(`FailedTestsTab` 색상·`TestResultDetailTableToolbar` 포맷)도 함께 바로잡았습니다.
* 동작 변화 없이 구조만 개선한 변경이 대부분으로, 이후 기능 추가·버그 수정의 안전성과 속도가 향상됩니다.
