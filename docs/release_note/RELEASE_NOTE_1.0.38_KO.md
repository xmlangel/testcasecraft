# 릴리스 노트 - v1.0.38

## 🚀 주요 변경 사항

### 📊 JUnit XML 파싱 고도화

- **구조화된 단계(Steps) 추출**: JUnit XML의 `<properties>` 내 `Step` 속성(파이프 구문) 및 커스텀 `<step>` 태그를 자동으로 분석하여 구조화된 테스트 단계 데이터를 생성합니다.
- **system-out 메타데이터 추출**: `<system-out>` 로그에 기록된 `[METADATA:...]` 형식을 분석하여 테스트 명세(설명, 단계, 기대/실제 결과)를 자동으로 복구하고 반영합니다.
- **범용 속성 지원**: 표준 JUnit 속성 외에도 XML에 포함된 모든 커스텀 속성을 인식하여 상세 패널에 표시합니다.
- **필드 매핑 개선**: `Description`, `ExpectedResult`, `ActualResult` 등 주요 속성을 시스템 필드와 자동으로 매핑하여 시인성을 높였습니다.

### 🎨 테스트 결과 UI 개선 (JUnit 상세 패널)

- **SQL/쿼리 포맷팅 강화**: 테스트 단계에 포함된 SQL 쿼리를 헤더에서 분리하고, 전용 코드 블록으로 렌더링하도록 개선했습니다.
- **가독성 및 줄바꿈 보존**: `white-space: pre-wrap`을 적용하여 긴 SQL 쿼리나 멀티라인 메시지가 원본 형식 그대로 표시되도록 수정했습니다.
- **정보 중복 제거**: 헤더에 표시된 핵심 정보가 'Execution Properties' 섹션에서 중복 표시되지 않도록 필터링 로직을 강화했습니다.

## 🛠 버그 수정 및 최적화

- **UI 레이아웃 조정**: 상세 패널 내의 정보 정렬을 최적화하여 좁은 화면에서도 데이터 확인이 용이하도록 개선했습니다.
- **파이프(|) 구분자 처리**: 단계(Step) 문자열이 파이프로 구분된 경우 이를 개별 단계로 분리하여 표시하는 로직을 추가했습니다.
