# Release Note - v1.0.78

## [1.0.78] - 2026-05-29

안녕하세요! 이번 1.0.78 업데이트에서는 **테스트케이스 작성 폼의 UX 를 전면 개선**하고, **사용자별 화면 환경설정을 서버에 영속화**하여 어떤 PC 에서 로그인해도 동일한 작업 환경을 즉시 복원하도록 하였습니다. 또한 트리·스프레드시트의 시각적 정돈, 풍부한 ShopFlow 샘플 데이터셋, 백엔드 동시성 안정화까지 한 사이클로 함께 묶었습니다.

### 주요 변경 사항

#### 🎨 테스트케이스 폼 레이아웃 전면 개선
* "테스트케이스 수정 / 생성" / "테스트 폴더 수정 / 생성" 4종 모드 타이틀(H6)을 제거하였습니다. 폼이 열린 것 자체가 모드를 의미하고, 우측 액션 버튼(수정·저장)과 displayId Chip 으로 충분히 식별 가능합니다.
* 모든 섹션을 **Accordion 으로 통일**하여 길이가 들쭉날쭉했던 폼 높이를 사용자가 직접 조절할 수 있게 하였습니다.
* 메인 입력 영역을 **2단 그리드**로 재배치하여 한 화면에서 더 많은 필드를 즉시 비교·편집할 수 있습니다.
* 메타데이터 영역을 **수정 가능 필드**와 **읽기 전용 필드**로 분리하고, UUID 는 잘리지 않고 전체 값을 표시하도록 변경하였습니다.
* 헤더 액션을 단일 행으로 통합하였습니다 — 버전 표시, RAG 등록, 일괄 저장이 한 줄에 정렬되어 한눈에 들어옵니다.
* `MarkdownFieldEditor` 가 입력 내용에 맞춰 **자동으로 높이를 조절**합니다 — 스크롤 안에서 길게 작성할 필요가 없습니다.

#### 🔘 필드 표시 토글 (FieldVisibilityMenu)
* 폼 우측 상단에 **필드 표시** 메뉴를 신설하였습니다. 사용자가 자주 쓰지 않는 메타 필드를 화면에서 제거해 본인 워크플로우에 맞춘 컴팩트 뷰를 만들 수 있습니다.
* 토글 가능한 9개 필드: 설명 · 사전 조건 · 사후 조건 · 자동화 여부 · Manual / Automation · 테스트 기법 · 우선순위 · 태그 · 연결된 RAG 문서.
* 핵심 필드(이름 · 스텝 · 기대 결과)는 토글 대상에서 제외되어 절대 사라지지 않습니다.

#### ☁️ 사용자별 UI 환경설정 서버 영속화
* 필드 표시 설정 · 입력 모드(개별 폼 vs 스프레드시트) 선택이 사용자별로 **서버에 영구 저장**됩니다.
* 다른 PC · 브라우저로 로그인해도 본인이 마지막에 설정한 화면이 그대로 복원됩니다.
* 같은 브라우저에서 사용자가 전환되어도 이전 사용자의 설정이 다음 사용자에게 누수되지 않도록 캐시를 사용자별 prefix 로 격리하였습니다.
* 변경 직후 페이지를 떠나도 저장이 보장됩니다 — `beforeunload` 와 `pagehide` 시점에 `fetch keepalive` 로 즉시 flush.

#### 🛡️ 백엔드 동시성·신뢰성 강화
* UI 환경설정 PATCH 가 동시에 여러 키로 발생해도 한쪽 변경이 손실되지 않도록 **PESSIMISTIC_WRITE 락 + `@Transactional`** 로 read-modify-write 를 직렬화하였습니다.
* `ObjectMapper` 인스턴스를 매 요청마다 새로 만들지 않고 **DI Bean 으로 재사용**하도록 통일하였습니다.

#### 🌲 테스트 케이스 트리 시각 개선
* 트리 노드의 displayId 가 `[TC-123]` 인라인 → **마지막 세그먼트만 monospace chip** 으로 표시되어 가독성이 크게 향상되었습니다. 호버 시 전체 ID 가 툴팁으로 노출됩니다.
* 노드 이름이 잘릴 때(컨테이너 폭 초과)만 호버 시 **전체 이름 툴팁**이 뜹니다 — 짧은 이름엔 불필요한 툴팁이 표시되지 않습니다.
* 트리 가상화에 `orderMap` 을 통합하여 **사용자 지정 순서 유지가 가상화·렌더링과 충돌하지 않도록** 처리하였습니다.

#### 📏 스프레드시트 셀 한글 30자 워드랩
* `TestCaseSpreadsheet` (react-spreadsheet) · `TestCaseDatasheetGrid` (MUI DataGrid) 두 입력 모드 모두에서, **한 셀의 콘텐츠가 한글 30자(약 480px)를 초과하면 자동으로 다음 줄로 워드랩** 됩니다.
* 긴 본문 때문에 컬럼이 옆으로 무한 확장되는 현상이 사라졌고, 행 높이가 콘텐츠에 맞춰 늘어납니다.
* 편집 모드(`<input>`)는 단일 라인 유지 — 표시 영역에서만 워드랩 적용.

#### 🛍️ ShopFlow 샘플 데이터셋 추가
* `scripts/shopflow_seed/` 에 **한 줄 명령으로 실제 운영 같은 전자상거래 SaaS 데모 데이터를 생성**하는 시드 스크립트를 추가하였습니다.
* 한국어(ShopFlow) / 영어(ShopFlow EN) 로케일을 독립적으로 또는 동시에 생성할 수 있습니다.
* 산출 규모(로케일별): 프로젝트 1 · 폴더 12 · 케이스 108 · 플랜 12 · 실행 12 (COMPLETED 6 / INPROGRESS 4 / NOTSTARTED 2) · 결과 134 · 자동화 PASS 35 / FAIL 20 · 담당자 distinct 5명 · JUnit 결과 3개(Cypress / Playwright / Jest).
* 멱등 재실행을 지원하여 시연 환경을 언제든 깨끗하게 복원할 수 있습니다.

#### 📘 사용자 매뉴얼 캡처 · 동기화 자동화
* `scripts/manual_capture.py` (Playwright Python) 로 매뉴얼이 참조하는 화면 스크린샷을 **한 번에 재캡처** 하고, 매뉴얼이 언급한 경로 vs 앱의 실제 라우트 커버리지를 감사합니다.
* 신규 페이지가 추가되면 감사가 누락을 검출하고, STEPS 와 매뉴얼 섹션을 함께 신설하도록 안내합니다 — **매뉴얼은 73장 고정이 아닌 점진적으로 성장하는 자산** 으로 다룹니다.
* `docs/manual/new/USER_MANUAL.md` 의 17 섹션 구조를 보존하면서 본 사이클의 UI 변경(폼 H6 제거, FieldVisibilityMenu, displayId chip, 셀 워드랩 등)을 박스 메모와 신규 소제목(§4-5 필드 표시 토글, §4-6 메타데이터 영역)으로 흡수하였습니다.

### 💡 사용자 조치 가이드

#### "필드 표시 토글" 사용법
1. 테스트케이스 폼 우측 상단의 **필드 표시** 메뉴를 클릭합니다.
2. 화면에서 가리고 싶은 메타 필드를 OFF 로 토글합니다 (예: 자동화 여부, 테스트 기법, 우선순위).
3. 설정이 즉시 반영되며, 다른 PC 에서 로그인해도 동일한 컴팩트 뷰가 자동 복원됩니다.

#### 변경분이 저장되었는지 확인
* 토글 / 입력 모드 변경 직후 화면 상단에 별도의 저장 인디케이터는 표시되지 않습니다. 약 0.5 초 이내에 서버에 비동기 저장되며, 페이지를 떠나도 안전합니다.
* 만약 네트워크 장애로 저장이 실패해도 다음 변경 시 자동으로 재시도되며, localStorage 캐시가 화면 표시를 유지합니다.
