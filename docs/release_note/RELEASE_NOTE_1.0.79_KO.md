# Release Note - v1.0.79

## [1.0.79] - 2026-05-29

안녕하세요! 이번 1.0.79 업데이트는 직전 1.0.78 폼 개선의 후속으로, **테스트케이스 입력 폼의 "테스트 단계"·"기대 결과" 에디터 높이를 내용에 맞춰 지능적으로 조절**하는 데 집중하였습니다. 비어 있을 때는 화면을 차지하지 않도록 최소화하고, 내용이 길어지면 최대 10줄까지 펼친 뒤 스크롤하여 한 화면에서 더 많은 케이스를 편하게 다룰 수 있습니다.

### 주요 변경 사항

#### 📐 테스트 단계 · 기대 결과 에디터 자동 높이
* **테스트 단계(Step / Expected)** 와 **전체 기대 결과** 마크다운 에디터가 기존 고정 높이(각 100px·120px)에서 **내용 기반 동적 높이**로 변경되었습니다.
* **내용이 없으면 최소 높이**(툴바 + 1줄)만 차지하여, 빈 스텝이 화면을 불필요하게 점유하지 않습니다.
* **내용이 있으면 줄 수만큼 늘어나되 최대 10줄에서 멈추고**, 그 이상은 에디터 내부에서 스크롤됩니다 — 긴 SQL·로그·시나리오를 입력해도 폼 전체가 끝없이 길어지지 않습니다.
* 테스트 단계 테이블에서 한 행의 **Step 칸과 Expected 칸 높이가 서로 정렬**되도록 두 필드 중 큰 값을 기준으로 높이를 맞췄습니다.
* 높이 계산 로직을 `utils/markdownEditorHeight.js` 공유 유틸(`computeMarkdownEditorHeight`)로 추출하여 두 컴포넌트가 동일한 규칙을 사용하도록 일원화하였습니다.
* `MarkdownFieldEditor` 의 `height` prop 은 이제 **선택적 최소 높이(floor)** 로 동작하여, 큰 에디터가 필요한 기존 화면(폴더 설명·사전/사후 조건 등)의 동작은 그대로 보존됩니다. `maxLines`(기본 10) prop 으로 화면별 상한을 조절할 수 있습니다.

#### 🛡️ 백엔드 컴파일 경고 정리
* UI 환경설정 병합 시 `objectMapper.readValue(existing, Map.class)` 가 raw 타입을 반환해 발생하던 `[unchecked]` 컴파일 경고를 **`TypeReference<Map<String, Object>>`** 로 교체하여 제거하였습니다.
* `./gradlew compileJava` 가 **경고 0건**으로 통과합니다.

### 💡 사용자 조치 가이드

#### 길어진 입력은 스크롤로 확인
* 테스트 단계나 기대 결과에 10줄을 초과하는 내용을 입력하면, 해당 에디터 영역 안에서 스크롤하여 전체 내용을 확인·편집할 수 있습니다.
* 빈 스텝을 추가하면 최소 높이로 표시되며, 입력을 시작하면 자동으로 늘어납니다 — 별도의 높이 조절 조작은 필요 없습니다.
