# Release Note - v1.0.91

## [1.0.91] - 2026-06-16

안녕하세요! 1.0.91은 **JIRA 연동과 테스트 실행 화면의 흐름**을 다듬은 업데이트입니다. 테스트 실패로 JIRA 이슈를 만들 때 테스트 단계가 함께 들어가도록 했고, 이슈에 연결된 테스트 이력이 보이지 않던 문제와, 목록에 필터를 건 뒤 상세에서 다음/이전을 누르면 필터를 무시하던 문제를 바로잡았습니다.

### 주요 변경 사항

#### 📝 JIRA 이슈 생성 시 테스트 단계 포함 (개선)

* 테스트 결과 입력 화면에서 **JIRA 이슈를 생성**할 때, 이슈 본문에 테스트 케이스·사전 조건·실제 결과만 들어가고 **테스트 단계가 빠지던** 문제를 개선했습니다.
* 이제 각 단계의 번호·동작·단계별 예상 결과가 `### 테스트 단계` 섹션으로 본문에 포함되어, 이슈만 봐도 재현 절차를 알 수 있습니다.

#### 🔗 JIRA 이슈 테스트 히스토리 누락 수정 (수정)

* JIRA 이슈에 **여러 개의 테스트 결과가 연결**되어 있는 경우, "JIRA 이슈 테스트 히스토리"에서 "연결된 테스트 결과가 없습니다"로 표시되던 문제를 수정했습니다.
* **원인**: 한 결과에 여러 이슈 키가 쉼표로 묶여 저장(예: `ONT-1086,ONT-904`)되는데, 조회는 정확히 일치하는 값만 찾아 키가 둘 이상이면 매칭되지 않았습니다.
* **수정**: 쉼표로 묶인 키 목록에서 **개별 키 단위로 매칭**하도록 조회를 바꿨습니다. 대시보드의 JIRA 상태 배지·상태 동기화에도 동일하게 적용됩니다. (부분 키 오인식 `ONT-1086` ↔ `ONT-10861` 도 방지)

#### 🧭 필터 적용 시 이전/다음 네비게이션 수정 (수정)

* 테스트 실행 목록에서 **필터(이름·상태 등)를 건 상태**로 항목 상세를 열고 "다음/이전"을 누르면, 필터된 목록이 아니라 **전체 목록의 다음 항목**으로 이동하던 문제를 수정했습니다.
* 이제 상세 다이얼로그와 전체화면 결과 뷰 **모두** 필터된 목록의 순서를 따라 이동합니다. (필터 정보가 없는 직접 링크 진입 시에는 기존처럼 전체 목록 기준으로 이동)

#### 🧪 내부 품질 (테스트·코드 정렬)

* 위 변경에 대한 회귀 테스트(백엔드·프런트엔드)를 추가했고, JIRA 이슈 본문 생성 로직에도 단위 테스트를 보강했습니다.
* 저장소 전체 Java 코드 포맷을 `google-java-format` 표준에 맞춰 일괄 정렬했습니다 (동작 변화 없음).
