# 테스트케이스 스프레드시트 UI 요소

이 문서는 `src/main/frontend/src/components/TestCase/TestCaseSpreadsheet.jsx` 파일을 기반으로 테스트케이스 스프레드시트 컴포넌트에 존재하는 주요 UI 요소를 식별하고 기록합니다.

## 1. 메인 카드 구조

- **`Card` 컴포넌트**
- **`CardContent`**

## 2. 헤더 영역

- **`Typography` (h6) - 테스트케이스 스프레드시트 제목**
  - 텍스트: `테스트케이스 스프레드시트`
  - Playwright 선택자 예시: `h6:has-text("테스트케이스 스프레드시트")`
- **행 수 (`Chip`)**: `X개 행`
- **스텝 수 (`Chip`)**: `X개 스텝`
- **변경됨 (`Chip`)**: `변경됨` (변경사항이 있을 때만 표시)
- **액션 버튼들**
  - **새로고침 버튼 (`Button` with `RefreshIcon`)**: `새로고침`
  - **행 추가 버튼 (`Button` with `AddIcon`)**: `행 추가`
  - **스텝 관리 메뉴 버튼 (`IconButton` with `SettingsIcon`)**: `⚙️` 아이콘
  - **일괄 저장 버튼 (`Button` with `SaveIcon`)**: `일괄 저장`

## 3. 사용법 안내

- **`Alert` 컴포넌트 (정보)**
  - 텍스트: `사용법: Excel과 같이 셀을 클릭하여 직접 편집하세요...`
  - 텍스트: `스텝 관리: ⚙️ 버튼을 클릭하여 스텝 수를 조정할 수 있습니다...`

## 4. 스프레드시트 그리드

- **`Spreadsheet` 컴포넌트**
  - 역할: 테스트케이스 데이터를 스프레드시트 형태로 표시하고 편집.
  - Playwright 선택자 예시: `div.Spreadsheet` (react-spreadsheet 라이브러리 내부 구조에 따라 다름)
  - **컬럼 라벨**: `이름`, `설명`, `사전조건`, `예상결과`, `Step X`, `Expected X` (동적 생성)
  - **셀**: 텍스트 필드 형태의 편집 가능한 셀.

## 5. 하단 정보

- **스텝 수 정보 (`Typography` caption)**: `* 현재 X개 스텝으로 설정되어 있습니다. 최대 10개 스텝까지 확장 가능합니다.`
- **미저장 경고 (`Typography` caption)**: `⚠️ 변경사항을 저장하지 않으면 손실될 수 있습니다.` (변경사항이 있을 때만 표시)

## 6. 스낵바 메시지

- **성공/에러 스낵바 (`Snackbar` 내 `Alert`)**: `저장되었습니다.`, `저장 중 오류가 발생했습니다: {{error}}`, `최신 데이터로 새로고침되었습니다.`, `새로고침 중 오류가 발생했습니다: {{error}}`, `스텝 수가 {{count}}개로 변경되었습니다.`

## 7. 스텝 관리 메뉴

- **`Menu` 컴포넌트**
- **스텝 추가 (`MenuItem` with `AddStepIcon`)**: `스텝 추가 (X개)`
- **스텝 제거 (`MenuItem` with `RemoveStepIcon`)**: `스텝 제거 (X개)`
- **스텝 수 직접 설정 (`MenuItem` with `SettingsIcon`)**: `스텝 수 직접 설정...`

## 8. 스텝 설정 다이얼로그

- **`Dialog` 컴포넌트**
- **다이얼로그 제목 (`DialogTitle`)**: `스텝 수 설정`
- **설명 (`Typography`)**: `테스트케이스의 스텝 수를 설정하세요. 기존 데이터는 유지됩니다.`
- **스텝 수 입력 (`TextField`)**: `스텝 수` 라벨, 숫자 입력 필드 (1~10).
- **취소 버튼 (`Button`)**: `취소`
- **적용 버튼 (`Button`)**: `적용`
